## Milestone Summary

**Key Accomplishments:**
1. cleanup.sh 共享库 — 11 个清理函数，覆盖 Docker/Node.js/文件三类残留
2. Jenkins 定期清理 Pipeline — cron 自动触发 + FORCE 手动跳过间隔
3. 所有服务部署后镜像清理闭环 — findclass-ssr/keycloak/noda-ops/nginx 全覆盖
4. nginx 蓝绿部署 DNS 修复 — resolver 127.0.0.11 + stop/rm/up 替代 --force-recreate
5. cleanup_dangling_images 计数修复 — 空结果不再触发 integer expression expected 错误

**Key Decisions:**
- 分步 prune 替代 docker system prune（细粒度控制 + 可追溯日志）
- build cache 保留 24h 热缓存（避免首次构建变慢）
- docker volume prune -f 不加 --all（保护 postgres_data 命名卷）
- pnpm store prune 每 7 天一次 + 标记文件避免与 install 冲突
- 清理 Pipeline 不包含 COMPOSE_BASE 和 DOPPLER_TOKEN（降低权限暴露）
- 清理 Pipeline 直接调用 cleanup.sh 函数，不依赖 pipeline-stages.sh 中间层

**Issues Resolved:**
- cleanup_dangling_images 空 grep 结果导致 integer expression expected 错误
- nginx --force-recreate 后 DNS 解析失败导致重启循环
- Alpine busybox wget localhost IPv6 解析问题（→ 127.0.0.1）
- Jenkinsfile.cleanup Groovy 变量在 sh 块中的语法错误

脚本路径常量：
```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "$SCRIPT_DIR/scripts/lib/log.sh"
SETUP_PG="$SCRIPT_DIR/scripts/setup-postgres-local.sh"
TOTAL_STEPS=4
```

实现以下 4 个函数：

1. **check_homebrew()** — 步骤 1/4：检查 Homebrew 是否安装
   - 使用 `command -v brew &>/dev/null` 检测
   - 已安装：`log_success` 显示版本（`brew --version | head -1`）
   - 未安装：`log_error "Homebrew 未安装"` + `log_info` 显示安装命令 `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"` + `exit 1`
   - 这个检查是幂等的：已安装则显示版本，不会重新安装

2. **install_postgresql()** — 步骤 2/4：安装 PostgreSQL + 创建开发数据库
   - `log_info "步骤 2/${TOTAL_STEPS}: 安装 PostgreSQL + 创建开发数据库"`
   - 检查 `$SETUP_PG` 文件是否存在，不存在则 `log_error` + `exit 1`（防御性检查）
   - 执行 `bash "$SETUP_PG" install`，直接透传其退出码
   - setup-postgres-local.sh install 内部已实现完整幂等（D-03）：
     - brew list 检查已安装则跳过
     - psql 检查数据库已存在则跳过
     - brew services 检查服务已运行则跳过

3. **verify_environment()** — 步骤 3/4：环境验证
   - `log_info "步骤 3/${TOTAL_STEPS}: 环境验证"`
   - 执行 `bash "$SETUP_PG" status`，其内部已实现 5 项检查（D-06）：
     - 检查 1/5: postgresql@17 安装状态 + 版本
     - 检查 2/5: brew services 运行状态
     - 检查 3/5: pg_isready 连接测试
     - 检查 4/5: noda_dev / keycloak_dev 数据库存在
     - 检查 5/5: 主版本号 17 匹配

4. **show_next_steps()** — 步骤 4/4：显示下一步指引
   - `log_info "步骤 4/${TOTAL_STEPS}: 下一步"`
   - 输出成功横幅：
     ```
     log_success "=========================================="
     log_success "开发环境就绪！"
     log_success "=========================================="
     ```
   - 提示信息：
     - `log_info "连接开发数据库: psql -d noda_dev"`
     - `log_info "查看 PG 状态: bash scripts/setup-postgres-local.sh status"`
     - `log_info "查看开发文档: docs/DEVELOPMENT.md"`

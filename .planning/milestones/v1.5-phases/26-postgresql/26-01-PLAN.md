**脚本头部：**
```bash
#!/bin/bash
set -euo pipefail
```

**常量定义：**
- `PG_FORMULA="postgresql@17"`（锁定版本，per D-05）
- `PG_PORT=5432`（默认端口，per D-02）
- `DEV_DATABASES=("noda_dev" "keycloak_dev")`（开发数据库列表，per D-06）
- `PG_DATA_DIR` 根据 `detect_homebrew_prefix` 动态设置为 `{prefix}/var/postgresql@17`

**detect_homebrew_prefix() 函数：**
```bash
detect_homebrew_prefix() {
  local arch
  arch=$(uname -m)
  if [ "$arch" = "arm64" ]; then
    echo "/opt/homebrew"
  else
    echo "/usr/local"
  fi
}
```

**usage() 函数：**
显示帮助信息，列出所有子命令：install / init-db / status / uninstall

**cmd_install() 函数（10 步）：**
1. 步骤 1/10: 架构检测 — 调用 `detect_homebrew_prefix`，`eval "$($HOMEBREW_PREFIX/bin/brew shellenv)"`
2. 步骤 2/10: 检查已安装 — `brew list postgresql@17 &>/dev/null`；如已安装，log_info 跳过
3. 步骤 3/10: 安装 postgresql@17 — `brew install postgresql@17`（per D-05 严格锁定版本）
4. 步骤 4/10: 链接二进制 — 检查 `command -v psql`，如不可用则 `brew link --force postgresql@17`
5. 步骤 5/10: 端口冲突检查 — `lsof -i :5432 &>/dev/null`；如被占用则 log_error 退出（per D-02 默认端口）
6. 步骤 6/10: 启动服务 — `brew services start postgresql@17`（per D-03 开机自启）；如已 started 则跳过
7. 步骤 7/10: 等待就绪 — 循环 `pg_isready`，最长等待 30 秒，每秒一次
8. 步骤 8/10: 验证 pg_hba.conf — 读取 `{PG_DATA_DIR}/pg_hba.conf`，检查 local 行和 host 127.0.0.1 行的认证方法是否为 trust；如果不是 trust，则自动替换为 trust（per D-01 trust 认证）。替换后执行 `brew services restart postgresql@17` 并重新等待就绪
9. 步骤 9/10: 创建开发数据库 — 调用 `cmd_init_db`（per D-06）
10. 步骤 10/10: 显示安装摘要 — 版本、端口、认证方式、数据库列表

**cmd_init_db() 函数：**
- 遍历 DEV_DATABASES 数组
- 对每个数据库：`psql -lqt 2>/dev/null | cut -d \| -f 1 | grep -qw "$db_name"` 检查是否存在
- 不存在则 `createdb "$db_name"`，存在则 log_info 跳过
- 幂等设计（per D-04）

**cmd_status() 函数（5 项检查）：**
1. 检查 1/5: 安装状态 — `brew list postgresql@17 &>/dev/null`；显示版本 `psql --version`
2. 检查 2/5: 服务状态 — `brew services list | grep "postgresql@17"`；检查是否 started（per D-03）
3. 检查 3/5: 连接测试 — `pg_isready &>/dev/null`
4. 检查 4/5: 开发数据库 — 检查 noda_dev 和 keycloak_dev 是否存在
5. 检查 5/5: 版本匹配 — `psql --version` 输出包含 "17."（per D-05）
- 最后汇总：所有通过显示 "所有检查通过"，否则 "部分检查未通过"

**cmd_uninstall() 函数（6 步）：**
1. 交互确认 — `read -r -p "确定要卸载 PostgreSQL 并删除所有数据？[y/N] "`，非 y 则退出
2. 停止服务 — `brew services stop postgresql@17`
3. 卸载包 — `brew uninstall postgresql@17`
4. 删除数据目录 — `rm -rf {PG_DATA_DIR}`（仅在确认后执行）
5. 清理 LaunchAgent — `brew services cleanup`
6. 显示完成信息

**子命令分发（文件末尾）：**
```bash
case "${1:-}" in
  install)    cmd_install "$@" ;;
  init-db)    cmd_init_db "$@" ;;
  status)     cmd_status "$@" ;;
  uninstall)  cmd_uninstall "$@" ;;
  *)          usage && exit 1 ;;
esac
```

**关键注意事项：**
- 所有 `brew` 和 `psql` 命令前需确保 Homebrew 环境已加载（`eval "$($HOMEBREW_PREFIX/bin/brew shellenv)"`）
- status 和 init-db 子命令也需要 detect_homebrew_prefix + eval brew shellenv
- pg_hba.conf 检查逻辑：用 grep 提取 local 和 host 127.0.0.1 行，验证 method 列（第二列）为 trust；如果不是，用 sed 替换。注意 macOS 的 pg_hba.conf 格式：`local   all   all   trust` 和 `host   all   all   127.0.0.1/32   trust`
- 确保 `chmod +x scripts/setup-postgres-local.sh`

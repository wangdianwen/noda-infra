# Milestone v1.1 — 基础设施现代化与文档更新

**Shipped:** 2026-04-11
**Status:** ✅ Shipped

---

## 概述

v1.0 备份系统交付后，对基础设施进行了全面的现代化改造：整合应用服务架构、修复 Keycloak Google 登录的关键 bug、拆分数据库为 prod/dev 双实例、清理历史遗留代码、以及全量文档更新。

---

## 关键成果

### 1. findclass-ssr 三合一服务

将前端静态文件 + API + SSR 渲染整合为单个 `findclass-ssr` 容器，替代旧的 web + api 双服务架构。

**变更：**
- 新增 `deploy/Dockerfile.findclass-ssr` 多阶段构建
- 新增 `docker/docker-compose.app.yml` 独立应用编排
- Nginx 路由简化为单一 upstream
- 移除旧的 `findclass-web` 和 `findclass-api` 服务定义

### 2. Keycloak Google 登录 5 层修复

修复了用户点击 Google 登录后被重定向到 `localhost:8080` 的复杂 bug，涉及 5 个嵌套层级：

| 层 | 问题 | 修复 |
|---|------|------|
| 1 | 前端 JS 硬编码 localhost:8080 | Dockerfile 添加 VITE_* ARG |
| 2 | Nginx /auth/ 代理覆盖应用路由 | 移除 /auth/ 到 Keycloak 的代理 |
| 3 | SSR 中间件跳过 /auth/callback | 移除跳过条件 |
| 4 | Keycloak v1 hostname 废弃选项 | 迁移到 v2 SPI |
| 5 | docker-compose 项目名冲突 | 统一为 noda-infra |

### 3. PostgreSQL prod/dev 双实例

将单一 PostgreSQL 容器拆分为生产和开发独立实例：

- **postgres (prod)**: `noda-infra-postgres-prod`，无端口暴露到宿主机
- **postgres-dev**: `noda-infra-postgres-dev`，暴露 5433:5432，带种子数据

同时更新了所有 40+ 处引用旧容器名 `noda-infra-postgres-1` 的脚本和配置。

### 4. noda-ops 容器合并

将运维相关的 `opdev`（备份）和 `cloudflared`（隧道）合并为单一 `noda-ops` 容器：

- 新增 `deploy/Dockerfile.noda-ops` 基于 Alpine + supervisord
- 同时运行 PostgreSQL 备份定时任务和 Cloudflare Tunnel
- 统一日志输出到 `/var/log/noda-backup/`

### 5. 全量文档更新

通过 gsd-docs-update 工作流重新生成 6 个核心文档，并通过 gsd-doc-verifier 修复审查队列中的 4 个文档：

- README.md: 修正目录树、补充缺失文档链接
- docs/architecture.md: 修正流量架构、服务端口、目录引用
- docs/CONFIGURATION.md: 修正环境变量名、添加缺失变量
- docs/GETTING-STARTED.md: 修正端口映射、部署命令
- docs/DEVELOPMENT.md: 补充备份系统 12 个工具库文档
- docs/TESTING.md: 修正容器名引用
- docs/DEPLOYMENT_GUIDE.md: 修正数据库列表、容器名
- docs/KEYCLOAK_SCRIPTS.md: 修正行号引用、容器名
- docs/README.md: 补充缺失文档条目

### 6. 基础设施清理

移除历史遗留代码和配置：

- 删除 Vercel OIDC Token 和 Supabase 引用（3 个文件）
- 删除 Jenkins 相关脚本（5 个文件，~637 行）
- 删除冗余目录（`backups/sql/`、旧 phase 目录）
- 删除过时设计文档（`docs/superpowers/`）

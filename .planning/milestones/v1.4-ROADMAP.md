# Milestone v1.4: CI/CD 零停机部署

**Status:** ✅ SHIPPED 2026-04-16
**Phases:** 19-25
**Total Plans:** 11

## Overview

Jenkins + 蓝绿部署实现编译失败不 down 站，自动回滚保护。从手动部署脚本迁移到 Jenkins Pipeline 驱动的零停机蓝绿部署流程。

## Phases

### Phase 19: Jenkins 安装与基础配置

**Goal**: 管理员可以在宿主机上安装、启动、获取初始密码、卸载 Jenkins，Jenkins 可直接操作 Docker
**Depends on**: Phase 18（前一里程碑，基础设施就绪）
**Plans**: 2 plans

Plans:

- [x] 19-01: setup-jenkins.sh 主脚本（7 个子命令：install/uninstall/status/show-password/restart/upgrade/reset-password）
- [x] 19-02: Jenkins init.groovy.d 自动化脚本（管理员/插件/安全/Pipeline 作业）+ 管理员凭据模板

**Details:**
- setup-jenkins.sh — Jenkins 完整生命周期管理脚本（7 个子命令）
- install 子命令 — Java 21 + Jenkins LTS + 端口 8888 + Docker 权限 + init.groovy.d 自动化配置
- uninstall 子命令 — 完全卸载（apt purge + 数据目录 + APT 源 + jenkins 用户）
- init.groovy.d 脚本实现 Jenkins 首次启动自动化配置（管理员用户、插件、安全策略、Pipeline 作业）
- 管理员凭据模板文件 jenkins-admin.env.example

### Phase 20: Nginx 蓝绿路由基础

**Goal**: nginx 的 findclass upstream 定义从 default.conf 抽离到独立 include 文件，Pipeline 可通过重写该文件 + reload 切换流量指向
**Depends on**: Phase 19（按顺序执行，避免并行部署冲突）
**Plans**: 1 plan

Plans:

- [x] 20-01: 抽离三个 upstream include 文件 + 验证 nginx reload 切换

**Details:**
- 3 个独立 upstream include 文件（snippets/upstream-*.conf）
- default.conf 不再包含 upstream 定义
- nginx 通过 snippets/*.conf 通配符自动加载 upstream
- 纯 upstream 块格式，Pipeline 直接重写整个文件

### Phase 24: Pipeline 增强特性

**Goal**: Pipeline 在部署前检查备份时效性，部署后自动清除 CDN 缓存和旧镜像，提升部署安全性和磁盘空间管理
**Depends on**: Phase 23（Pipeline 基础流程可用）
**Plans**: 2 plans

Plans:

- [x] 24-01: pipeline-stages.sh 三项增强函数（check_backup_freshness + pipeline_purge_cdn + cleanup_old_images 时间阈值重写）
- [x] 24-02: Jenkinsfile 新增 CDN Purge stage（9 阶段 Pipeline）

**Details:**

- CDN Purge stage 位于 Verify 之后、Cleanup 之前
- withCredentials 注入 Cloudflare 凭据，单引号 sh 块防止凭据泄露

## Milestone Summary

**Key Decisions:**

- Decision: Jenkins 宿主机原生安装（Rationale: Docker-in-Docker 安全风险大）
- Decision: Upstream include 抽离（Rationale: Pipeline 可重写文件切换流量）
- Decision: 蓝绿容器 docker run 管理（Rationale: 避免与 compose 冲突）
- Decision: Pipeline 手动触发（Rationale: 生产环境安全控制）
- Decision: Jenkinsfile Declarative Pipeline（Rationale: 可读性和可维护性）
- Decision: envsubst 指定变量替换（Rationale: 避免替换 shell 内置变量）
- Decision: CDN 清除非阻止型（Rationale: 部署不应因 CDN API 问题失败）
- Decision: 时间阈值镜像清理替代计数保留（Rationale: 更符合磁盘管理需求）
- Decision: sh 块使用单引号（Rationale: 防止 Groovy 插值泄露凭据到构建日志）

**Issues Resolved:**

- manage-containers.sh source guard 保护（避免函数重复定义）
- pipeline-stages.sh 独立封装解决无 source guard 问题
- pipeline_preflight 增强为完整环境检查，降低运维排障成本

**Issues Deferred:**

- Prisma 7 兼容性迁移（依赖 noda-apps 仓库变更）
- 部署通知（邮件/Slack/Discord webhook）
- 参数化构建支持

**Technical Debt Incurred:**

- 无重大技术债务

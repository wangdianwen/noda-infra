# Quick Task 260410-al7: 修复Keycloak登录跳转到8080端口的bug

## 问题分析

Keycloak 生成 URL 时使用端口 8080 而非 443，导致 Google OAuth 登录跳转到 `https://auth.noda.co.nz:8080/...`。

**根因：** `KC_HOSTNAME_STRICT: "false"` 允许 Keycloak 从请求中派生端口值。Cloudflare Tunnel 转发请求到 `http://keycloak:8080`，Keycloak 从请求中获取端口 8080 并覆盖了配置的 `KC_HOSTNAME_PORT: "443"`。

**修复：** 设置 `KC_HOSTNAME_STRICT: "true"`，强制 Keycloak 始终使用配置的 hostname 和 port。

## Tasks

### Task 1: 修复 docker-compose.yml Keycloak 配置
- **files:** docker/docker-compose.yml
- **action:** 将 `KC_HOSTNAME_STRICT` 从 `"false"` 改为 `"true"`
- **verify:** 配置值已更新
- **done:** KC_HOSTNAME_STRICT 为 "true"

### Task 2: 修复 docker-compose.prod.yml Keycloak 配置
- **files:** docker/docker-compose.prod.yml
- **action:** 将 `KC_HOSTNAME_STRICT` 从 `"false"` 改为 `"true"`
- **verify:** 配置值已更新
- **done:** KC_HOSTNAME_STRICT 为 "true"

### Task 3: 修复部署脚本使用 prod override
- **files:** scripts/deploy/deploy-infrastructure-prod.sh
- **action:** 将 `docker-compose -f docker/docker-compose.yml` 改为 `docker-compose -f docker/docker-compose.yml -f docker/docker-compose.prod.yml`
- **verify:** 所有 docker-compose 命令使用两个文件
- **done:** 部署脚本同时使用 base 和 prod 配置

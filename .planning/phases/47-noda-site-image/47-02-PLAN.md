**Build 阶段修改 -- 在 pipeline_build 调用前复制 nginx 配置文件到构建上下文：**

将 Build stage 的 sh 块从：
```groovy
sh '''
    source scripts/lib/log.sh
    source scripts/pipeline-stages.sh
    pipeline_build "$WORKSPACE/noda-apps" "$GIT_SHA"
'''
```
修改为：
```groovy
sh '''
    source scripts/lib/log.sh
    # 将 nginx 配置文件复制到构建上下文（Dockerfile COPY 路径相对于 noda-apps）
    mkdir -p "$WORKSPACE/noda-apps/deploy/nginx"
    cp "$WORKSPACE/deploy/nginx/nginx.conf" "$WORKSPACE/noda-apps/deploy/nginx/nginx.conf"
    cp "$WORKSPACE/deploy/nginx/default.conf" "$WORKSPACE/noda-apps/deploy/nginx/default.conf"
    source scripts/pipeline-stages.sh
    pipeline_build "$WORKSPACE/noda-apps" "$GIT_SHA"
    # 清理构建上下文中的临时文件
    rm -rf "$WORKSPACE/noda-apps/deploy"
'''
```

关键点（per RESEARCH.md Open Question 1，推荐方案三）：
- Dockerfile 中 `COPY deploy/nginx/nginx.conf` 的路径是相对于 docker build 的构建上下文（$WORKSPACE/noda-apps）
- 但 nginx 配置文件实际在 noda-infra 仓库的 deploy/nginx/ 中
- 解决方案：构建前复制到 noda-apps/deploy/nginx/，构建后清理

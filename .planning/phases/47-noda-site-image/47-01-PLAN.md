创建 deploy/nginx/ 目录及两个配置文件。

**deploy/nginx/nginx.conf（主配置）：**
- `pid /tmp/nginx.pid;` -- PID 文件指向 /tmp（tmpfs 可写）
- `error_log /tmp/error.log notice;` -- 错误日志指向 /tmp
- `events { worker_connections 1024; }`
- `http` 块内容：
  - `include /etc/nginx/mime.types;` -- nginx 自带 MIME 映射
  - `default_type application/octet-stream;`
  - `access_log /tmp/access.log;` -- 访问日志指向 /tmp
  - 所有 temp_path 指向 /tmp：`client_body_temp_path /tmp/client_temp;`、`proxy_temp_path /tmp/proxy_temp_path;`、`fastcgi_temp_path /tmp/fastcgi_temp;`、`uwsgi_temp_path /tmp/uwsgi_temp;`、`scgi_temp_path /tmp/scgi_temp;`
  - `include /etc/nginx/conf.d/*.conf;`
- **不写 `user` 指令** -- Docker `USER nginx` 已以 nginx 用户身份运行进程，nginx.conf 的 user 指令会尝试切换用户导致权限冲突（per RESEARCH.md Pitfall 5）
- **不添加 gzip 配置** -- Cloudflare + 外层 nginx 已处理压缩（per D-03）
- **不添加缓存头** -- 外层 default.conf 已处理缓存策略（per D-02）

**deploy/nginx/default.conf（server 块）：**
```nginx
server {
    listen 3000;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```
- `listen 3000` -- 保持蓝绿部署兼容（per D-01）
- `try_files $uri $uri/ /index.html` -- SPA fallback，等价于 serve -s（per D-01）
- 不添加 gzip、缓存头、access_log 自定义 -- 全部由外层 nginx 和 Cloudflare 处理

**Stage 1 (builder) -- 完全保留，不修改任何内容：**
保持 FROM node:20-alpine AS builder 及所有后续命令不变。

**Stage 2 (runner) -- 替换为以下内容：**
```dockerfile
# ----------------------------------------
# Stage 2: nginx 轻量运行时
# ----------------------------------------
FROM nginx:1.25-alpine AS runner

# 移除默认 nginx 配置
RUN rm /etc/nginx/conf.d/default.conf

# 复制自定义 nginx 配置（非 root + 日志/PID 指向 /tmp）
COPY deploy/nginx/nginx.conf /etc/nginx/nginx.conf
COPY deploy/nginx/default.conf /etc/nginx/conf.d/default.conf

# 仅复制构建产物
COPY --from=builder /app/apps/site/dist /usr/share/nginx/html

# 设置目录权限（Phase 48 可优化为 COPY --chown）
RUN chown -R nginx:nginx /usr/share/nginx/html /var/cache/nginx

USER nginx

EXPOSE 3000

# 健康检查（注意：使用 127.0.0.1 而非 localhost，BusyBox wget IPv6 问题 per RESEARCH Pitfall 1）
HEALTHCHECK --interval=10s --timeout=3s --retries=3 --start-period=3s \
  CMD wget --quiet --tries=1 --spider http://127.0.0.1:3000/ || exit 1

CMD ["nginx", "-g", "daemon off;"]
```

关键点：
- 基础镜像 `nginx:1.25-alpine` -- 与外层 nginx 版本一致，可共享层缓存
- 移除默认配置 `RUN rm /etc/nginx/conf.d/default.conf` -- 避免冲突
- nginx 配置文件 COPY 路径是相对于构建上下文的（即 noda-apps 目录），但 pipeline_build() 中 nginx 配置文件需要复制到构建上下文中（这将在 Plan 02 的 pipeline 适配中处理，通过在 pipeline_build 中添加预复制步骤）
- `USER nginx` -- nginx:1.25-alpine 自带 nginx 用户 (UID=101)，不需要 adduser
- `chown -R nginx:nginx` -- 确保 nginx 用户可读静态文件和可写缓存目录
- HEALTHCHECK 使用 `127.0.0.1` 而非 `localhost` -- BusyBox wget IPv6 解析问题（per RESEARCH Pitfall 1）
- start_period 设为 3s -- nginx 启动 < 1 秒（per D-08）
- 不安装 curl -- 使用 nginx:alpine 自带的 BusyBox wget（更轻量）
  </action>

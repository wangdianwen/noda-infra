  <how-to-verify>
    ## 步骤 1：注册 Doppler 账号

    1. 访问 https://dashboard.doppler.com/register
    2. 注册 Developer Free 账号（无需信用卡）
    3. 确认邮箱

    ## 步骤 2：创建项目

    1. 在 Doppler Dashboard 点击 "Create Project"
    2. 项目名：`noda`
    3. 环境保持默认 `prod`（或创建 `prod` 环境）
    4. 确认项目创建成功

    ## 步骤 3：导入密钥

    方式 A — Dashboard UI 手动添加（推荐）：
    1. 进入 noda 项目 → prod config → Secrets
    2. 逐个添加以下 15 个密钥（值从服务器上的 docker/.env 和 .env.production 文件中获取）：

    **从 docker/.env 获取（基础设施密钥，7 个）：**
    - POSTGRES_USER
    - POSTGRES_PASSWORD
    - POSTGRES_DB
    - KEYCLOAK_ADMIN_USER
    - KEYCLOAK_ADMIN_PASSWORD
    - KEYCLOAK_DB_PASSWORD
    - CLOUDFLARE_TUNNEL_TOKEN

    **额外从 docker/.env 获取（应用密钥，2 个）：**
    - ANTHROPIC_AUTH_TOKEN
    - ANTHROPIC_BASE_URL

    **从 .env.production 获取（应用密钥，6 个）：**
    - SMTP_HOST
    - SMTP_PORT
    - SMTP_FROM
    - SMTP_USER
    - SMTP_PASSWORD
    - RESEND_API_KEY

    **注意重叠密钥：** 7 个基础设施密钥在两个文件中都有，确认值一致即可。Doppler 中只存一份。

    方式 B — 使用 CLI 从服务器上传：
    ```bash
    # 在 Jenkins 宿主机上执行
    export DOPPLER_TOKEN='dp.st.prd.xxxx'  # 先完成步骤 4 获取 token
    # 合并两个 .env 文件并去重
    cat /path/to/noda-infra/docker/.env /path/to/noda-infra/.env.production | sort -u -t= -k1,1 > /tmp/noda-secrets.env
    # 手动编辑 /tmp/noda-secrets.env 删除不需要的行（空值、VITE_*、注释等）
    doppler secrets upload /tmp/noda-secrets.env --project noda --config prod
    rm -f /tmp/noda-secrets.env
    ```

    ## 步骤 4：创建 Service Token

    1. 进入 noda 项目 → Access → Service Tokens
    2. 点击 "Generate Service Token"
    3. Config 选择：`prod`
    4. Access 选择：`Read`（只读，最小权限原则）
    5. Name: `jenkins`
    6. **立即复制生成的 token（格式：dp.st.prd.xxx），此值只显示一次！**

    ## 步骤 5：存储 Service Token 到 Jenkins Credentials

    1. 访问 Jenkins UI（http://服务器IP:8888）
    2. Manage Jenkins → Credentials → System → Global credentials
    3. Add Credentials：
       - Kind: Secret text
       - Scope: Global
       - Secret: 粘贴刚才复制的 Service Token（dp.st.prd.xxx）
       - ID: `DOPPLER_TOKEN`
       - Description: `Doppler Service Token (read-only, prod)`
    4. 点击 Create

    ## 步骤 6：存储 Service Token 到密码管理器

    将 Service Token 明文存储到你的密码管理器（1Password/Bitwarden 等），标签为 "Doppler - noda - jenkins Service Token"。

    ## 步骤 7：验证

    在 Jenkins 宿主机上执行验证脚本：
    ```bash
    export DOPPLER_TOKEN='dp.st.prd.xxx'  # 使用刚才创建的 token
    bash scripts/verify-doppler-secrets.sh
    ```
    期望输出："验证通过: 15/15 密钥完整"

    也可以手动验证：
    ```bash
    doppler secrets download --format=env --no-file --project noda --config prod
    ```
    期望输出：包含所有 15 个密钥的 KEY=VALUE 格式内容

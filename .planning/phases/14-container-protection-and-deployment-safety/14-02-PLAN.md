**2. 重写 config/nginx/conf.d/default.conf，添加 upstream 块和错误页面配置：**

在文件最顶部（`map` 指令之前）添加 upstream 定义：

```nginx
# ============================================
# Upstream 定义（故障转移）
# ============================================
upstream keycloak_backend {
    server keycloak:8080 max_fails=3 fail_timeout=30s;
}

upstream findclass_backend {
    server findclass-ssr:3001 max_fails=3 fail_timeout=30s;
}
```

**修改 Keycloak server 块：**
将 `proxy_pass http://keycloak:8080/;` 改为 `proxy_pass http://keycloak_backend;`（注意去掉末尾斜杠）。

在 Keycloak server 块的 `location / { ... }` 块内，`include /etc/nginx/snippets/proxy-websocket.conf;` 之前添加：
```nginx
        proxy_next_upstream error timeout http_502 http_503;
        proxy_next_upstream_timeout 5s;
        proxy_next_upstream_tries 2;
```

在 Keycloak server 块的 `location /` 块之后，`server` 块结束之前添加：
```nginx
    # 自定义错误页
    error_page 502 503 /50x.html;
    location = /50x.html {
        root /etc/nginx/errors;
        internal;
    }
```

**修改主应用 server 块：**
将 `proxy_pass http://findclass-ssr:3001;` 改为 `proxy_pass http://findclass_backend;`。

在主应用 server 块的 `location / { ... }` 块内，`include /etc/nginx/snippets/proxy-common.conf;` 之前添加：
```nginx
        proxy_next_upstream error timeout http_502 http_503;
        proxy_next_upstream_timeout 5s;
        proxy_next_upstream_tries 2;
```

在主应用 server 块的 `location /health { ... }` 块之后，`server` 块结束之前添加：
```nginx
    # 自定义错误页
    error_page 502 503 /50x.html;
    location = /50x.html {
        root /etc/nginx/errors;
        internal;
    }
```

**3. 在 docker/docker-compose.yml 中为 nginx 挂载 errors 目录：**

在 nginx 服务的 volumes 列表中（`../config/nginx/snippets:/etc/nginx/snippets:ro` 行之后）添加：
```yaml
      - ../config/nginx/errors:/etc/nginx/errors:ro
```

**注意事项：**
- `proxy_next_upstream_timeout 5s` 和 `proxy_next_upstream_tries 2` 自 Nginx 1.7.5 起支持，nginx:1.25-alpine 完全兼容
- upstream 中只有单台 server，`proxy_next_upstream` 在单 server 场景下不会触发重试到其他 server，但会处理连接错误后快速返回错误页面
- error_page 的 `internal` 指令确保用户不能直接请求 /50x.html 路径
- `/etc/nginx/errors` 路径需要通过 volume 挂载（见第 3 步）

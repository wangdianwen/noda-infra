创建 `scripts/backup/lib/health.sh` 健康检查库文件，实现以下功能：

**函数清单：**
1. `check_postgres_connection()` - 检查 PostgreSQL 连接状态
2. `check_disk_space()` - 检查磁盘空间（容器 + 宿主机）
3. `get_database_size()` - 获取单个数据库大小
4. `get_total_database_size()` - 获取所有数据库总大小
5. `check_prerequisites()` - 综合检查函数（连接 + 磁盘空间）

**实现细节：**

**check_postgres_connection():**
- 使用 `docker exec noda-infra-postgres-1 pg_isready -U postgres`
- 返回码：0=成功，非0=失败
- 失败时输出错误信息并返回退出码 1

**check_disk_space():**
- 查询所有数据库大小：`pg_database_size(datname)`
- 计算所需空间：数据库总大小 × 2（D-15）
- 检查容器内磁盘：`docker exec noda-infra-postgres-1 df -BG /var/lib/postgresql/data`
- 检查宿主机磁盘：`df -BG /var/lib/docker/volumes/noda-infra_postgres_data/_data`
- 可用空间必须 >= 所需空间，否则返回退出码 1

**get_database_size(db_name):**
- 使用 `psql -U postgres -d postgres -t -c "SELECT pg_database_size('$db_name');"`
- 返回字节数

**get_total_database_size():**
- 查询 `pg_database` 系统表，排除模板数据库（datistemplate = false）
- 累加所有用户数据库大小
- 排除系统数据库（postgres、template0、template1）

**错误处理：**
- 使用 `set -euo pipefail` 确保错误传播
- 定义退出码常量：
  ```bash
  EXIT_SUCCESS=0
  EXIT_CONNECTION_FAILED=1
  EXIT_DISK_SPACE_INSUFFICIENT=2
  ```
- 所有错误消息使用符号前缀（❌ 连接失败、⚠️ 磁盘空间不足）

**文件头注释：**
```bash
#!/bin/bash
# ============================================
# Noda 数据库备份系统 - 健康检查库
# ============================================
# 功能：备份前前置检查（连接状态 + 磁盘空间）
# 依赖：config.sh（加载配置）
# ============================================
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 宿主机 → PostgreSQL 容器 | docker exec 跨容器边界执行命令 |
| 配置文件 → 脚本 | 环境变量从文件加载到脚本进程 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | PostgreSQL 连接检查 | mitigate | pg_isready 验证真实的 PostgreSQL 服务响应，不依赖虚假端口开放 |
| T-01-02 | Tampering | 配置文件篡改 | mitigate | .env.backup 文件权限设置为 600，验证配置值合法性（端口号范围、路径格式） |
| T-01-03 | Information Disclosure | 配置文件泄露 | accept | Phase 1 仅使用模板，实际配置由运维人员管理，不在代码库中存储敏感信息 |
| T-01-04 | Denial of Service | 磁盘空间耗尽 | mitigate | 备份前检查磁盘空间（数据库大小 × 2），空间不足时拒绝执行并返回明确错误 |
| T-01-05 | Elevation of Privilege | docker exec 权限 | accept | 脚本假设运行在有权限访问 Docker 的用户环境，Phase 1 不处理权限隔离 |

## Security Notes

- **ASVS V5（输入验证）**：配置项验证（端口号、路径格式、保留天数范围）
- **ASVS V6（加密）**：.pgpass 文件权限 600，避免密码泄露（Phase 1 不实现，预留接口）
- **退出码标准化**：遵循 MONITOR-05（0=成功、1=连接失败、2=备份失败等）
</threat_model>

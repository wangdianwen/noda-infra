- D-13: 备份文件权限设置为严格的 600（仅所有者可读写）
- D-16: 备份失败时清理所有已创建的备份文件

- D-34: 使用 `.pgpass` 文件管理数据库密码（避免硬编码）

```bash
cleanup_files() {
  local file_list=$1
  for file in $file_list; do
    if [ -f "$file" ]; then

    fi
  done
}
```

```bash
calculate_checksum() {
  local file_path=$1
  sha256sum "$file_path" | awk '{print $1}'
}
```

**字节格式化函数：**
```bash
format_bytes() {
  local bytes=$1
  if [ $bytes -lt 1024 ]; then
    echo "${bytes}B"
  elif [ $bytes -lt 1048576 ]; then
    echo "$((bytes / 1024))KB"
  elif [ $bytes -lt 1073741824 ]; then
    echo "$((bytes / 1048576))MB"
  else
    echo "$((bytes / 1073741824))GB"
  fi
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 脚本 → PostgreSQL 容器 | docker exec 跨容器边界执行备份命令 |
| 备份文件 → 文件系统 | 备份文件写入 Docker volume 映射的宿主机目录 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | 备份文件篡改 | mitigate | 使用 SHA-256 校验和验证备份完整性，文件权限 600 防止未授权修改 |
| T-02-02 | Information Disclosure | 备份文件泄露 | mitigate | 备份文件权限 600（仅所有者可读写），避免敏感数据泄露 |
| T-02-03 | Denial of Service | 磁盘空间耗尽 | mitigate | 备份失败时自动清理已创建的备份文件（D-16），避免留下不完整的备份占用空间 |
| T-02-04 | Spoofing | 数据库欺骗 | accept | 通过 pg_database 系统表自动发现数据库，排除模板数据库，不依赖用户输入 |
| T-02-05 | Elevation of Privilege | pg_dump 权限 | accept | 脚本假设运行在有权限执行 docker exec 和访问 PostgreSQL 的环境 |

## Security Notes

- **ASVS V6（加密）**：备份文件权限 600，SHA-256 校验和验证完整性
- **ASVS V5（输入验证）**：数据库名称从系统表查询，不接受用户输入，避免注入攻击
- **错误处理**：遇到错误立即失败并清理，避免留下不完整的备份文件
</threat_model>

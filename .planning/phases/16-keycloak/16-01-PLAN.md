<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Cloudflare CDN | 外部不可信流量，TLS 终止点 |
| Cloudflare → noda-ops (Tunnel) | 已认证 Tunnel 连接，内部通信 |
| noda-ops → noda-nginx | Docker 内部网络，HTTP 明文 |
| noda-nginx → keycloak:8080 | Docker 内部网络，HTTP 明文，新增代理层 |
| 宿主机 → keycloak | 移除（原 8080/9000 端口暴露已消除） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Information Disclosure | keycloak ports 暴露 | mitigate | D-03 移除所有 ports 映射，keycloak 仅 Docker 内部网络可达 |
| T-16-02 | Tampering | Cloudflare → nginx → keycloak 中间人 | mitigate | Cloudflare TLS 终止 + Docker 内部网络隔离；nginx 添加 X-Forwarded 安全头 |
| T-16-03 | Spoofing | healthcheck 伪造 | accept | healthcheck 在容器内执行，攻击者需容器级访问；威胁面极低 |
| T-16-04 | Information Disclosure | Keycloak Admin Console 暴露 | mitigate | Admin Console 通过 auth.noda.co.nz 访问，受 Keycloak 认证保护；nginx 添加 X-Frame-Options: SAMEORIGIN |
| T-16-05 | Denial of Service | nginx 代理层单点 | accept | 单节点部署，已配置 upstream failover（max_fails=3, fail_timeout=30s, proxy_next_upstream） |
</threat_model>

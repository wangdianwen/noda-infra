创建 `scripts/setup-jenkins.sh`，实现以下功能：

**脚本头部：**
```bash
#!/bin/bash
set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
source "$PROJECT_ROOT/scripts/lib/log.sh"

JENKINS_PORT=8888
JENKINS_HOME="/var/lib/jenkins"
JENKINS_LOG="/var/log/jenkins"
JENKINS_OVERRIDE_DIR="/etc/systemd/system/jenkins.service.d"
JENKINS_OVERRIDE_CONF="$JENKINS_OVERRIDE_DIR/override.conf"
GROOVY_SRC_DIR="$SCRIPT_DIR/jenkins/init.groovy.d"
ADMIN_ENV_TEMPLATE="$SCRIPT_DIR/jenkins/config/jenkins-admin.env.example"
```

**usage() 函数 — 列出所有 7 个子命令及简要说明。**

**wait_for_jenkins() 辅助函数：**
- 轮询 `curl -sf "http://localhost:${JENKINS_PORT}/login"` 等待 Jenkins 就绪
- 最大等待 120 秒，每 5 秒一次
- 使用 log_info/log_success/log_error 报告状态

**cmd_install() — 安装 Java 21 + Jenkins LTS + Docker 权限（per D-01, D-02, D-04）：**
1. 检查是否已安装（`dpkg -l jenkins`），已安装则提示并退出
2. 安装 Java 21: `sudo apt update && sudo apt install -y fontconfig openjdk-21-jre`
3. 验证 Java: `java -version`，失败则退出
4. 添加 Jenkins apt 源:
   ```bash
   sudo wget -O /etc/apt/keyrings/jenkins-keyring.asc \
     https://pkg.jenkins.io/debian-stable/jenkins.io-2026.key
   echo "deb [signed-by=/etc/apt/keyrings/jenkins-keyring.asc]" \
     https://pkg.jenkins.io/debian-stable binary/ | \
     sudo tee /etc/apt/sources.list.d/jenkins.list > /dev/null
   sudo apt update
   ```
5. 安装 Jenkins: `sudo apt install -y jenkins`
6. 配置端口 8888 (per D-01) — 创建 systemd override:
   ```bash
   sudo mkdir -p "$JENKINS_OVERRIDE_DIR"
   sudo tee "$JENKINS_OVERRIDE_CONF" > /dev/null <<'EOF'
[Service]
Environment="JENKINS_PORT=8888"
Environment="JAVA_OPTS=-Djava.awt.headless=true -Djenkins.install.runSetupWizard=false"
EOF
   sudo systemctl daemon-reload
   ```
7. 写入 init.groovy.d 脚本（如果 `$GROOVY_SRC_DIR` 目录存在）:
   ```bash
   sudo mkdir -p "$JENKINS_HOME/init.groovy.d"
   sudo cp "$GROOVY_SRC_DIR"/*.groovy "$JENKINS_HOME/init.groovy.d/"
   sudo chown -R jenkins:jenkins "$JENKINS_HOME/init.groovy.d"
   ```
8. 写入管理员凭据（如果 `$JENKINS_HOME/.admin.env` 不存在且 `$ADMIN_ENV_TEMPLATE` 的实际配置文件存在）:
   - 检查 `$SCRIPT_DIR/jenkins/config/jenkins-admin.env`（实际凭据文件，不入 Git）
   - 如果存在，`sudo cp` 到 `$JENKINS_HOME/.admin.env` 并 `chown jenkins:jenkins`
9. Docker 权限 (per JENK-03): `sudo usermod -aG docker jenkins`
10. 启动 Jenkins: `sudo systemctl enable jenkins && sudo systemctl start jenkins`
11. 等待 Jenkins 就绪: `wait_for_jenkins`
12. 清理 init.groovy.d（首次配置完成后删除）: `sudo rm -rf "$JENKINS_HOME/init.groovy.d"`
13. 输出安装成功信息和访问地址

**cmd_uninstall() — 完全卸载 (per D-03)：**
1. 停止服务: `sudo systemctl stop jenkins 2>/dev/null || true`
2. 禁用服务: `sudo systemctl disable jenkins 2>/dev/null || true`
3. 卸载包: `sudo apt remove --purge -y jenkins`
4. 删除数据目录: `sudo rm -rf "$JENKINS_HOME"`（/var/lib/jenkins）
5. 删除日志目录: `sudo rm -rf "$JENKINS_LOG"`（/var/log/jenkins）
6. 删除缓存: `sudo rm -rf /var/cache/jenkins`
7. 删除 APT 源: `sudo rm -f /etc/apt/sources.list.d/jenkins.list`
8. 删除 keyring: `sudo rm -f /etc/apt/keyrings/jenkins-keyring.asc`
9. 删除 systemd override: `sudo rm -rf "$JENKINS_OVERRIDE_DIR"`
10. `sudo systemctl daemon-reload`
11. jenkins 用户从 docker 组移除: `sudo gpasswd -d jenkins docker 2>/dev/null || true`
12. 删除 jenkins 系统用户: `sudo userdel jenkins 2>/dev/null || true`
13. `sudo apt autoremove -y`
14. 输出卸载成功信息

**cmd_status() — 状态检查：**
1. 检查 Jenkins 包是否安装（`dpkg -l jenkins`）
2. 检查服务状态（`systemctl is-active jenkins`）
3. 检查监听端口（`ss -tlnp | grep $JENKINS_PORT` 或 `curl -sf http://localhost:$JENKINS_PORT/login`）
4. 检查 Docker 权限（`groups jenkins | grep docker`）
5. 检查 Jenkins 版本（`jenkins --version` 或从 dpkg 获取）
6. 汇总输出所有状态

**case 分发到 7 个子命令（install, uninstall, status, show-password, restart, upgrade, reset-password）。** 其中 show-password、restart、upgrade、reset-password 在 Task 2 实现，此处先放占位函数（只有 log_info 提示"见后续版本"）。

**cmd_show_password() — 获取初始管理员密码 (per JENK-04)：**
1. 检查 `/var/lib/jenkins/secrets/initialAdminPassword` 是否存在
2. 如果存在，`sudo cat` 输出内容
3. 如果不存在，提示可能已经完成初始设置，建议使用 reset-password

**cmd_restart() — 重启 Jenkins：**
1. `sudo systemctl restart jenkins`
2. 调用 `wait_for_jenkins` 等待就绪
3. 输出重启成功信息

**cmd_upgrade() — 升级到最新 LTS：**
1. `sudo apt update`
2. `sudo apt install --only-upgrade jenkins`
3. `sudo systemctl restart jenkins`
4. 调用 `wait_for_jenkins` 等待就绪
5. 输出升级后版本号

**cmd_reset_password() — 重置管理员密码：**
1. 接受参数 `$2` 作为新密码，为空则报错退出
2. 写入临时 Groovy 脚本到 `/tmp/reset-jenkins-password.groovy`:
   ```groovy
   import jenkins.model.*
   import hudson.security.*
   def instance = Jenkins.getInstance()
   def realm = instance.getSecurityRealm()
   def user = realm.getUser(adminUser)
   if (user != null) {
       user.setPassword(newPassword)
   }
   instance.save()
   ```
   其中 adminUser 默认 "admin"，newPassword 从参数读取
3. 使用 `sudo -u jenkins java -jar /usr/share/java/jenkins.war` 执行 groovy 脚本（或使用 `jenkins-cli.jar` 从 `$JENKINS_HOME/war/WEB-INF/jenkins-cli.jar` 获取）
4. 实际使用方式: `sudo bash -c "cd $JENKINS_HOME && echo 'groovy script content' | java -jar /usr/share/java/jenkins-cli.jar -s http://localhost:$JENKINS_PORT/ groovy"`
5. 清理临时脚本
6. 输出密码重置成功信息

注意：reset-password 的实现路径可能因 Jenkins 版本而异。如果 jenkins-cli.jar 不在预期路径，脚本应检查多个可能位置并报错提示。

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 管理员 → setup-jenkins.sh | 脚本需要 sudo 权限执行，信任边界在 sudo 提权 |
| setup-jenkins.sh → Jenkins 服务 | 脚本创建 Jenkins 服务和配置，控制 Jenkins 运行时行为 |
| jenkins 用户 → Docker socket | jenkins 用户通过 docker 组获得 Docker daemon 完全控制权 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Elevation of Privilege | jenkins 用户加入 docker 组 | accept | jenkins 用户仅通过 docker 组权限操作 Docker（宿主机安装，非容器，攻击面较小）。Pipeline 限制执行预定义脚本，非任意命令 |
| T-19-02 | Tampering | reset-password 无密码 sudo 执行 groovy | mitigate | reset-password 子命令要求传入密码参数，不使用默认密码；groovy 脚本执行后立即清理临时文件 |
| T-19-03 | Information Disclosure | Jenkins 监听 0.0.0.0:8888 | accept | Jenkins 仅用于内部 CI/CD，不暴露到公网（Cloudflare Tunnel 不代理 8888 端口）；后续可通过 iptables 限制仅 localhost |
| T-19-04 | Spoofing | 管理员凭据存储在明文 .admin.env | mitigate | .admin.env 文件权限设为 600（仅 jenkins 用户可读）；模板文件 jenkins-admin.env.example 入 Git，实际凭据文件在 .gitignore 中排除 |
</threat_model>

文件头注释:
```
// 统一基础设施 Jenkins Pipeline
// 7 阶段: Pre-flight -> Backup -> Human Approval -> Deploy -> Health Check -> Verify -> Cleanup
// 手动触发（无自动触发配置）
// 参数化服务选择: keycloak / nginx / noda-ops / postgres
```

**pipeline 块结构:**

```groovy
pipeline {
    agent any

    environment {
        PROJECT_ROOT = "${WORKSPACE}"
        COMPOSE_BASE = "-f docker/docker-compose.yml -f docker/docker-compose.prod.yml"
        BACKUP_HOST_DIR = "${WORKSPACE}/docker/volumes/backup"
    }

    options {
        disableConcurrentBuilds()
        buildDiscarder(logRotator(numToKeepStr: '20'))
        timestamps()
    }
```

**parameters 块 (per D-01):**
```groovy
    parameters {
        choice(
            name: 'SERVICE',
            choices: ['keycloak', 'nginx', 'noda-ops', 'postgres'],
            description: '选择要部署的基础设施服务'
        )
    }
```

**stages 块 — 7 个阶段:**

**Stage 1: Pre-flight**
```groovy
stage('Pre-flight') {
    steps {
        sh '''
            source scripts/lib/log.sh
            source scripts/pipeline-stages.sh
            pipeline_infra_preflight "$SERVICE"
        '''
    }
}
```

**Stage 2: Backup (条件化, per D-06)**
使用 `when` 仅对 keycloak/postgres 执行:
```groovy
stage('Backup') {
    when {
        anyOf {
            expression { params.SERVICE == 'keycloak' }
            expression { params.SERVICE == 'postgres' }
        }
    }
    steps {
        sh '''
            source scripts/lib/log.sh
            source scripts/pipeline-stages.sh
            pipeline_backup_database "$SERVICE"
        '''
    }
}
```

**Stage 3: Human Approval (条件化, per D-09)**
仅对 postgres 生效，30 分钟超时:
```groovy
stage('Human Approval') {
    when {
        expression { params.SERVICE == 'postgres' }
    }
    steps {
        timeout(time: 30, unit: 'MINUTES') {
            input message: """确认重启 PostgreSQL?
            服务: ${params.SERVICE}
            影响范围: Keycloak、findclass-ssr、noda-ops 将短暂断连数据库
            备份文件: 请查看上方 Backup 阶段日志
            """.stripIndent(), ok: '确认重启'
        }
    }
}
```

**Stage 4: Deploy**
```groovy
stage('Deploy') {
    steps {
        sh '''
            source scripts/lib/log.sh
            source scripts/pipeline-stages.sh
            pipeline_infra_deploy "$SERVICE"
        '''
    }
}
```

注意: keycloak 的 Deploy 会调用 keycloak-blue-green-deploy.sh，该脚本内部包含 Pull Image -> Deploy -> Health Check -> Switch -> E2E Verify 完整流程。但 Jenkinsfile.infra 仍然独立定义 Health Check 和 Verify 阶段，keycloak 的 pipeline_infra_health_check 做二次验证。

**Stage 5: Health Check**
```groovy
stage('Health Check') {
    steps {
        sh '''
            source scripts/lib/log.sh
            source scripts/pipeline-stages.sh
            pipeline_infra_health_check "$SERVICE"
        '''
    }
}
```

**Stage 6: Verify**
```groovy
stage('Verify') {
    steps {
        sh '''
            source scripts/lib/log.sh
            source scripts/pipeline-stages.sh
            pipeline_infra_verify "$SERVICE"
        '''
    }
}
```

**Stage 7: Cleanup**
```groovy
stage('Cleanup') {
    steps {
        sh '''
            source scripts/lib/log.sh
            source scripts/pipeline-stages.sh
            pipeline_infra_cleanup "$SERVICE"
        '''
    }
}
```

**post 块:**
```groovy
post {
    failure {
        script {
            sh '''
                source scripts/lib/log.sh
                source scripts/pipeline-stages.sh
                pipeline_infra_failure_cleanup "$SERVICE"
            '''
            archiveArtifacts artifacts: 'deploy-failure-*.log', allowEmptyArchive: true
        }
    }
    success {
        echo "基础设施服务 ${params.SERVICE} 部署成功"
    }
}
```

关键设计决策:
1. 所有 sh 块使用单引号（'''），防止 Groovy 插值泄露凭据到日志（per Phase 24 决策）
2. 不包含 CDN Purge 阶段 — 基础设施服务不涉及静态资源缓存
3. 不包含 Build/Test 阶段 — 基础设施服务不从源码构建（keycloak 用官方镜像，nginx/noda-ops 用 compose）
4. Keycloak 的 Pull Image 由 keycloak-blue-green-deploy.sh 内部处理
5. SERVICE 变量通过 Jenkins parameters 传入，在 sh 块中作为环境变量 "$SERVICE" 使用

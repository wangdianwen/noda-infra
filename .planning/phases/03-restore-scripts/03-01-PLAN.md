## Objective

创建完整的数据库恢复系统，允许运维人员从 Backblaze B2 云存储列出、下载和恢复 PostgreSQL 数据库备份。支持安全的测试恢复（恢复到不同数据库名）和完整的验证流程。

## Tasks

### Task 1: 创建核心恢复库

<read_first>
- `scripts/backup/lib/cloud.sh` - Reuse rclone config functions (setup_rclone_config, cleanup_rclone_config)
- `scripts/backup/lib/config.sh` - Use B2 configuration getters (get_b2_bucket_name, get_b2_path)
- `scripts/backup/lib/log.sh` - Use logging functions
</read_first>

<action>
创建 `scripts/backup/lib/restore.sh` 文件，实现以下函数：

**1. list_backups_b2() - 列出 B2 备份**
- 使用 rclone ls 列出 B2 bucket 中的所有文件
- 解析文件名格式：`dbname_YYYYMMDD_HHMMSS.sql` 或 `.dump`
- 格式化输出：日期时间、数据库名、文件大小（MB）、文件名
- 按时间排序显示

**2. download_backup() - 下载备份文件**
- 验证文件名格式（使用正则表达式）
- 使用 rclone copy 从 B2 下载指定文件
- 验证下载文件完整性
- 返回本地文件路径

**3. restore_database() - 恢复数据库**
- 删除目标数据库（如果存在）
- 创建新数据库
- 根据文件扩展名选择恢复方法：
  - `.dump` 文件：使用 pg_restore -j 4
  - `.sql` 文件：使用 psql -f
- 恢复前提示用户确认（显示目标数据库和备份文件）
- 恢复后验证：查询表数量

**4. verify_backup_integrity() - 验证备份完整性**
- 检查文件存在性
- 检查文件大小（> 100 bytes）
- 验证文件格式：
  - `.dump` 文件：运行 pg_restore -l
  - `.sql` 文件：检查是否包含 "PostgreSQL database dump"

所有函数必须：
- 使用 log_info, log_error, log_success, log_warn 记录操作
- 返回 0 表示成功，非 0 表示失败
- 在失败时提供明确的错误信息
</action>

<acceptance_criteria>
- `scripts/backup/lib/restore.sh` exists and is executable
- File contains function list_backups_b2 with rclone ls implementation
- File contains function download_backup with rclone copy implementation
- File contains function restore_database with pg_restore/psql logic
- File contains function verify_backup_integrity with format validation
- All functions use logging functions from lib/log.sh
- All functions return 0 on success, non-zero on failure
</acceptance_criteria>

### Task 2: 创建主恢复脚本

<read_first>
- `scripts/backup/backup-postgres.sh` - Follow the same structure and style
- `scripts/backup/lib/restore.sh` - Use the restore library functions
- `scripts/backup/lib/constants.sh` - Use exit code constants
</read_first>

<action>
创建 `scripts/backup/restore-postgres.sh` 文件，实现命令行接口：

**1. 参数解析**
- `-l, --list-backups`: 列出所有可用备份
- `-r, --restore <filename>`: 恢复指定备份文件
- `-d, --database <name>`: 恢复到指定数据库名（用于测试）
- `-o, --output-dir <dir>`: 下载备份到指定目录
- `-v, --verify`: 仅验证备份完整性
- `-h, --help`: 显示帮助信息

**2. 主流程 (main 函数)**
- 加载配置：source lib/config.sh, load_config
- 解析命令行参数
- 根据操作类型执行：
  - `list`: 调用 list_backups_b2()
  - `restore`: 调用 download_backup() → verify_backup_integrity() → restore_database()
  - `verify`: 调用 download_backup() → verify_backup_integrity()
- 恢复完成后询问是否删除下载的文件（如果不是用户指定的目录）

**3. 错误处理**
- 未指定备份文件时：提示使用 --list-backups
- 下载失败时：退出码 EXIT_RESTORE_FAILED
- 恢复失败时：退出码 EXIT_RESTORE_FAILED
- 无效参数时：退出码 EXIT_INVALID_ARGS

**4. 帮助文档**
包含完整的用法说明、选项列表、示例和环境变量说明
</action>

<acceptance_criteria>
- `scripts/backup/restore-postgres.sh` exists and is executable (chmod +x)
- Script supports --list-backups option
- Script supports --restore <filename> option
- Script supports --database <name> option
- Script supports --verify option
- Script supports --output-dir <dir> option
- Script shows help with --help option
- Script returns EXIT_RESTORE_FAILED (7) on restore failures
- Script returns EXIT_INVALID_ARGS (8) on invalid arguments
</acceptance_criteria>

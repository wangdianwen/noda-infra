    完整服务定义：
    ```yaml
      # ----------------------------------------
      # Keycloak 开发环境（独立容器）
      # start-dev 模式：自动禁用主题缓存、放宽 HTTPS 要求
      # ----------------------------------------
      keycloak-dev:
        image: quay.io/keycloak/keycloak:26.2.3
        container_name: noda-infra-keycloak-dev
        labels:
          - "noda.service-group=infra"
        restart: unless-stopped
        ports:
          - "127.0.0.1:18080:8080"  # HTTP 开发端口（仅本机，per T-12-01 安全缓解）
          - "127.0.0.1:19000:9000"  # 管理端口（仅本机）
        environment:
          # 数据库：连接 postgres-dev 的 keycloak_dev（与生产完全隔离）
          KC_DB: postgres
          KC_DB_URL: jdbc:postgresql://postgres-dev:5432/keycloak_dev
          KC_DB_USERNAME: ${POSTGRES_USER}
          KC_DB_PASSWORD: ${POSTGRES_PASSWORD}
          # 启用 HTTP（start-dev 自动处理，显式声明确保）
          KC_HTTP_ENABLED: "true"
          # 管理员账号
          KEYCLOAK_ADMIN: ${KEYCLOAK_ADMIN_USER}
          KEYCLOAK_ADMIN_PASSWORD: ${KEYCLOAK_ADMIN_PASSWORD}
        # start-dev：自动禁用主题缓存 + 放宽安全限制（per D-01）
        command: start-dev
        # 读写挂载主题目录（per D-06, D-07）
        volumes:
          - ./services/keycloak/themes:/opt/keycloak/themes/noda
        networks:
          - noda-network
        depends_on:
          postgres-dev:
            condition: service_healthy
        # 健康检查（与 prod 模式一致：TCP 9000 端口）
        healthcheck:
          test: ["CMD-SHELL", "echo > /dev/tcp/localhost/9000 2>/dev/null || exit 1"]
          interval: 30s
          timeout: 10s
          retries: 3
          start_period: 60s
    ```

    关键设计决策说明：
    - 端口绑定 `127.0.0.1`（不是 `0.0.0.0`）防止开发环境暴露在公网（T-12-01 安全缓解）
    - KC_DB_URL 严格指向 `postgres-dev:5432/keycloak_dev`，不是生产数据库（T-12-02 安全缓解）
    - 不设置 KC_HOSTNAME / KC_PROXY / KC_PROXY_HEADERS（start-dev 自动处理，否则 localhost 访问异常）
    - 不配置 Google OAuth（per D-05），start-dev 默认开启密码认证
    - 不设置 JVM 内存限制（Claude's discretion，开发环境资源消耗低，观察后再决定）

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 宿主机 -> keycloak-dev 容器 | 端口 18080/19000 映射，限制 127.0.0.1 绑定 |
| keycloak-dev -> postgres-dev | KC_DB_URL 跨容器数据库连接 |
| 宿主机文件系统 -> keycloak-dev 容器 | 主题目录读写挂载 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Information Disclosure | keycloak-dev 端口 18080/19000 | mitigate | 端口绑定 `127.0.0.1:18080:8080` 和 `127.0.0.1:19000:9000`，仅本机可访问，不暴露到公网。与生产 keycloak 在 docker-compose.prod.yml 中使用 `127.0.0.1:8080:8080` 模式一致 |
| T-12-02 | Tampering | KC_DB_URL 数据库连接 | mitigate | KC_DB_URL 严格使用 `postgres-dev:5432/keycloak_dev`，代码审查确认不指向生产数据库。keycloak_dev 数据库通过 init-dev SQL 独立创建 |
| T-12-03 | Elevation of Privilege | KEYCLOAK_ADMIN_PASSWORD 复用 | accept | 开发环境与生产复用相同管理员密码（环境变量引用）。风险可接受：端口仅绑定 localhost，开发环境不在公网暴露。如需更高隔离可在 .env 中设置独立 DEV_KEYCLOAK_ADMIN_PASSWORD 变量 |
| T-12-04 | Tampering | 主题目录读写挂载 | accept | Docker 挂载使用相对路径 `./services/keycloak/themes`，已限制在项目目录内。路径遍历风险由 Docker 引擎处理。开发环境需要写入能力（per D-06） |
</threat_model>

**cmd_migrate_data() 函数实现：**

1. **Homebrew 环境加载** — 调用 `detect_homebrew_prefix` + `eval "$($HOMEBREW_PREFIX/bin/brew shellenv)"`

2. **前置检查 — 本地 PG 是否运行：**
   - 运行 `pg_isready &>/dev/null`
   - 如未就绪则 log_error "本地 PostgreSQL 未运行，请先执行 install" 并 exit 1

3. **前置检查 — Docker 容器是否运行：**
   - 运行 `docker ps --format "{{.Names}}" | grep -q "noda-infra-postgres-dev"`
   - 如未运行：
     - log_warn "postgres-dev 容器未运行，无法迁移"
     - 交互询问 `read -r -p "跳过迁移？[Y/n] "`，默认 Y 跳过
     - 如跳过则 log_info "跳过数据迁移" 并 return 0

4. **迁移前状态显示：**
   - 显示 Docker 容器内每个数据库的大小
   - `docker exec noda-infra-postgres-dev psql -U postgres -c "SELECT datname, pg_size_pretty(pg_database_size(datname)) FROM pg_database WHERE datname IN ('noda_dev', 'keycloak_dev');"`

5. **定义迁移数据库列表：**
   - `MIGRATE_DATABASES=("noda_dev" "keycloak_dev")`（与 DEV_DATABASES 一致，per D-06）

6. **逐数据库迁移 — migrate_database() 内部函数：**
   ```bash
   migrate_database() {
     local db_name="$1"
     local dump_file="/tmp/${db_name}_dump_$$.sql"

     log_info "导出 ${db_name} 从 Docker 容器..."

     # 使用 Docker 容器内的 pg_dump（版本 17.9 完全匹配，per D-03）
     docker exec noda-infra-postgres-dev pg_dump -U postgres -d "$db_name" \
       --no-owner --no-privileges > "$dump_file"

     if [ ! -s "$dump_file" ]; then
       log_error "${db_name} 导出失败（空文件）"
       rm -f "$dump_file"
       return 1
     fi

     log_info "导入 ${db_name} 到本地 PostgreSQL..."

     # 幂等：先删除再重建
     psql -d postgres -c "DROP DATABASE IF EXISTS ${db_name};" 2>/dev/null || true
     psql -d postgres -c "CREATE DATABASE ${db_name};"
     psql -d "$db_name" < "$dump_file"

     # 清理临时文件
     rm -f "$dump_file"
     log_success "${db_name} 迁移完成"
   }
   ```

7. **执行迁移循环：**
   ```bash
   local success_count=0
   local fail_count=0
   for db in "${MIGRATE_DATABASES[@]}"; do
     if migrate_database "$db"; then
       ((success_count++))
     else
       ((fail_count++))
     fi
   done
   ```

**关键注意事项：**
- `pg_dump` 使用 `--no-owner --no-privileges` 确保导入时不依赖原始用户角色（per D-03）
- dump 文件使用 `$$` 后缀避免并发冲突
- 如果 pg_dump 在 Docker 容器内执行失败（如数据库不存在），log_warn 而非 log_error，继续迁移其他数据库


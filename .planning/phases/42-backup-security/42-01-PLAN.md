3. **替换上传逻辑**（第 104-119 行的 `if [[ "$DRY_RUN" == "false" ]]; then` 块）：将 b2 upload-file 替换为 rclone copy。容器内 rclone 配置已由 entrypoint-ops.sh 创建（RCLONE_CONFIG 环境变量），本地执行则使用临时配置：

```bash
if [[ "$DRY_RUN" == "false" ]]; then
    info "上传到 Backblaze B2..."
    info "Bucket: $B2_BUCKET | 路径: $B2_REMOTE_PATH"

    # 容器内使用 entrypoint-ops.sh 配置的 rclone，本地使用临时配置
    local_rclone_config=""
    if [[ -z "${RCLONE_CONFIG:-}" ]] || [[ ! -f "${RCLONE_CONFIG:-}" ]]; then
        # 本地执行：创建临时 rclone 配置（复用 cloud.sh 模式）
        if [[ -z "${B2_ACCOUNT_ID:-}" ]] || [[ -z "${B2_APPLICATION_KEY:-}" ]]; then
            error "B2_ACCOUNT_ID 和 B2_APPLICATION_KEY 环境变量未设置"
            exit 1
        fi
        local_rclone_config=$(mktemp)
        chmod 600 "$local_rclone_config"
        cat >"$local_rclone_config" <<EOF
[b2remote]
type = b2
account = $B2_ACCOUNT_ID
key = $B2_APPLICATION_KEY
EOF
    fi

    RCLONE_FLAGS=("--log-level" "INFO")
    if [[ -n "$local_rclone_config" ]]; then
        RCLONE_FLAGS+=("--config" "$local_rclone_config")
    fi

    if rclone copy "$OUTPUT_FILE" "b2remote:${B2_BUCKET}/doppler-backup/" \
        "${RCLONE_FLAGS[@]}"; then
        info "上传成功"
    else
        error "rclone 上传失败"
        rm -f "$local_rclone_config"
        warn "加密文件仍保留在本地: $OUTPUT_FILE"
        exit 1
    fi

    # 清理临时 rclone 配置
    rm -f "$local_rclone_config"
else
    info "[dry-run] 跳过 B2 上传"
    info "[dry-run] 加密文件保留在: $OUTPUT_FILE"
fi
```

4. **更新帮助信息**：将 `--config` 默认值说明从 `prod` 改为 `prd`（保持与 Doppler 实际配置名一致，第 34 行已有默认值 `CONFIG="prd"` 但帮助文本写的是 `prod`）。

注意：保留所有其他功能不变（age 加密、--dry-run 支持、参数解析、错误处理）。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker Compose 环境变量 → noda-ops 容器 | DOPPLER_TOKEN 通过环境变量注入，在容器进程内存中可读 |
| noda-ops 容器 → Doppler Cloud API | 容器内 doppler CLI 通过 HTTPS 调用 Doppler API |
| noda-ops 容器 → Backblaze B2 | rclone 通过 HTTPS 上传加密快照 |
| 明文密钥管道 | doppler secrets download 输出通过管道直接加密，不落盘 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-01 | Information Disclosure | DOPPLER_TOKEN 环境变量 | mitigate | docker-compose.yml 使用 ${DOPPLER_TOKEN} 从宿主机环境变量注入，不硬编码值 |
| T-42-02 | Information Disclosure | 密钥明文在内存中 | mitigate | doppler secrets download 通过管道直接输入 age 加密，明文不写入文件系统 |
| T-42-03 | Information Disclosure | B2 上传凭据 | accept | B2 凭据已通过 docker-compose 环境变量注入，与数据库备份共享同一信任边界 |
| T-42-04 | Tampering | 加密快照被篡改 | accept | age 公钥加密 + B2 不可变存储，篡改风险极低 |
| T-42-05 | Denial of Service | Doppler API 宕机导致备份失败 | accept | 下次 cron 周期自动重试；密钥变更频率低，单次失败不影响恢复能力 |
</threat_model>

  **背景（per D-04, D-06）：**
  为剩余 3 个 compose 文件添加分组标签。注意：docker-compose.dev.yml 是 overlay 文件，只定义覆盖配置。根据 Docker Compose 合并规则，overlay 中的服务如果不设置 labels，会继承基础文件中的 labels。

  **具体修改 — docker/docker-compose.dev.yml：**

  此文件是 overlay，定义了 postgres、postgres-dev、nginx、cloudflared、keycloak、findclass-ssr 的覆盖配置。

  - `postgres-dev` 服务是新服务（不在基础 yml 中），需要添加标签：
    ```yaml
    labels:
      - "noda.service-group=infra"
    ```

  - `keycloak` 服务：overlay 中覆盖了 environment 和 ports 和 healthcheck。由于 Docker Compose 合并 labels 时 overlay 的 labels 会替换基础文件的 labels，需要保留基础文件中的标签：
    ```yaml
    labels:
      - "noda.service-group=infra"
    ```

  - `findclass-ssr` 服务：overlay 中覆盖了 ports 和 environment。同样需要保留标签：
    ```yaml
    labels:
      - "noda.service-group=apps"
    ```

  - `nginx` 服务：只覆盖了 ports 和 volumes，不设置 labels 不影响继承。

  - `cloudflared` 服务：使用了 profiles（开发环境不启动），不设置 labels。

  - `postgres` 服务：只覆盖了注释，不设置 labels 不影响继承。

  **具体修改 — docker/docker-compose.simple.yml：**

  此文件是独立完整配置（不作为 overlay 使用），所有服务都需要添加标签。

  为以下 5 个服务各添加 `labels` 指令：

  1. `postgres` 服务：
     ```yaml
     labels:
       - "noda.service-group=infra"
     ```

  2. `postgres-dev` 服务：
     ```yaml
     labels:
       - "noda.service-group=infra"
     ```

  3. `nginx` 服务：
     ```yaml
     labels:
       - "noda.service-group=infra"
     ```

  4. `keycloak` 服务：
     ```yaml
     labels:
       - "noda.service-group=infra"
     ```

  5. `cloudflared` 服务：
     ```yaml
     labels:
       - "noda.service-group=infra"
     ```

  **具体修改 — docker/docker-compose.dev-standalone.yml：**

  此文件是独立开发环境配置，只有 postgres-dev 服务。

  1. `postgres-dev` 服务：
     ```yaml
     labels:
       - "noda.service-group=infra"
     ```

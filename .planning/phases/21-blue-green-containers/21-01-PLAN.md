**脚本头部**（严格遵循 setup-jenkins.sh 模式）：
```bash
#!/bin/bash
set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

source "$PROJECT_ROOT/scripts/lib/log.sh"

```

**常量定义**：
```bash
ACTIVE_ENV_FILE="/opt/noda/active-env"
NGINX_CONTAINER="noda-infra-nginx"
UPSTREAM_CONF="$PROJECT_ROOT/config/nginx/snippets/upstream-findclass.conf"
ENV_TEMPLATE="$PROJECT_ROOT/docker/env-findclass-ssr.env"
NETWORK_NAME="noda-network"
IMAGE_NAME="findclass-ssr"
```

**辅助函数**：

1. `validate_env(env)` — 验证参数必须是 `blue` 或 `green`，否则报错退出
2. `get_active_env()` — 读取 `$ACTIVE_ENV_FILE`，文件不存在时返回 `blue`
3. `get_inactive_env()` — 返回活跃环境的互补值（blue<->green）
4. `set_active_env(env)` — 原子写入 `$ACTIVE_ENV_FILE`（先写 .tmp 再 mv）
5. `prepare_env_file(env)` — 读取 `$ENV_TEMPLATE`，使用 `envsubst` 替换 `${POSTGRES_USER}`、`${POSTGRES_PASSWORD}`、`${RESEND_API_KEY}` 三个变量，输出到 `/tmp/findclass-ssr-${env}.env`，返回临时文件路径。**安全措施**：只替换这三个变量，避免替换 $HOSTNAME 等 shell 内置变量（per RESEARCH Pitfall 6）。如果 `envsubst` 不可用则报错提示安装 `gettext`
6. `get_container_name(env)` — 返回 `findclass-ssr-${env}`
7. `is_container_running(name)` — `docker inspect -f '{{.State.Running}}' $name 2>/dev/null || echo false`

**核心函数 `run_container(env, image)`**（D-03: Phase 22 通过 source 调用此函数）：

组装 docker run 命令，参数逐项从 docker-compose.app.yml 翻译（per D-10, D-11, D-12）：

```bash
docker run -d \
  --name "findclass-ssr-${env}" \
  --network noda-network \
  --restart unless-stopped \
  --stop-timeout 30 \
  --security-opt no-new-privileges \
  --cap-drop ALL \
  --read-only \
  --tmpfs /tmp \
  --memory 512m \
  --memory-reservation 128m \
  --cpus 1 \
  --log-driver json-file \

  --health-interval 30s \
  --health-timeout 10s \
  --health-retries 3 \
  --health-start-period 60s \
  "$image"
```

**子命令实现**：

1. **`cmd_init()`**（D-07 完整迁移流程）：
   - 步骤 1：检测 compose 管理的 findclass-ssr 容器是否存在（`docker inspect findclass-ssr`），获取当前镜像名
   - 步骤 2：如果不存在，检查 `/opt/noda/active-env` 是否已存在（说明已初始化），如果也没有则报错退出
   - 步骤 3：如果找到 compose 容器，提示 "init 将短暂中断服务（约 60-90 秒）"，等待用户确认
   - 步骤 4：停止 compose 容器：`docker stop findclass-ssr && docker rm findclass-ssr`
   - 步骤 5：使用 compose 容器的相同镜像调用 `run_container blue "$current_image"`
   - 步骤 6：等待健康检查：`wait_container_healthy findclass-ssr-blue 120`
   - 步骤 7：更新 nginx upstream：写入 `upstream findclass_backend { server findclass-ssr-blue:3001 max_fails=3 fail_timeout=30s; }` 到 `$UPSTREAM_CONF`
   - 步骤 8：reload nginx：`docker exec $NGINX_CONTAINER nginx -s reload`
   - 步骤 9：写入状态文件：`set_active_env blue`
   - 步骤 10：打印成功信息

2. **`cmd_start()`**（启动指定环境容器）：
   - 参数：`$2` = env（blue 或 green）
   - 验证参数（`validate_env`）
   - 检查容器是否已运行（`is_container_running`），已运行则提示并退出
   - 检查 `noda-network` 是否存在：`docker network inspect noda-network >/dev/null 2>&1`
   - 默认镜像为 `findclass-ssr:latest`（可通过 `$3` 覆盖）
   - 调用 `run_container "$env" "$image"`
   - 等待健康检查：`wait_container_healthy "findclass-ssr-${env}" 120`

3. **`cmd_stop()`**（停止指定环境容器）：
   - 参数：`$2` = env
   - 验证参数
   - 安全检查：如果要停止的是当前活跃环境，警告并确认（`log_warn` + `read -p`）
   - `docker stop -t 30 "findclass-ssr-${env}"`
   - `docker rm "findclass-ssr-${env}"`

4. **`cmd_restart()`**（重启指定环境容器）：
   - 参数：`$2` = env
   - 获取当前容器镜像：`docker inspect --format='{{.Config.Image}}' "findclass-ssr-${env}"`
   - `cmd_stop` 然后 `cmd_start`（使用相同镜像）

5. **`cmd_status()`**（显示蓝绿状态）：
   - 读取活跃环境：`get_active_env`
   - 对 blue 和 green 分别检查：
     - 容器是否存在（`docker inspect`）
     - 运行状态（Running）
     - 健康状态（Health.Status）
     - 镜像名（Config.Image）
     - 创建时间（Created）
   - 打印格式化输出，活跃环境用标记 `[ACTIVE]`

6. **`cmd_logs()`**（查看容器日志）：
   - 参数：`$2` = env
   - 验证参数
   - `docker logs "findclass-ssr-${env}" ${@:3}`（支持 --tail、--follow 等额外参数透传）

7. **`cmd_switch()`**（切换活跃环境）：
   - 参数：`$2` = target env（blue 或 green）
   - 验证参数
   - 检查目标容器是否在运行且健康：`docker inspect` + `wait_container_healthy "findclass-ssr-${target}" 10`
   - 如果目标容器不健康，报错退出（不切换）
   - 更新 upstream 文件：写入 `server findclass-ssr-${target}:3001 max_fails=3 fail_timeout=30s;`
   - nginx -t 验证配置：`docker exec $NGINX_CONTAINER nginx -t`
   - nginx reload：`docker exec $NGINX_CONTAINER nginx -s reload`
   - 更新状态文件：`set_active_env "$target"`
   - 打印切换结果

8. **`usage()`**（帮助信息）：
   - 显示所有子命令及其说明，格式与 setup-jenkins.sh 的 usage 一致

**子命令分发**（case 语句，与 setup-jenkins.sh 一致）：
```bash
case "${1:-}" in
  init)    cmd_init "$@" ;;
  start)   cmd_start "$@" ;;
  stop)    cmd_stop "$@" ;;
  restart) cmd_restart "$@" ;;
  status)  cmd_status "$@" ;;
  logs)    cmd_logs "$@" ;;
  switch)  cmd_switch "$@" ;;
  *)       usage && exit 1 ;;
esac
```

- 所有 `docker exec nginx` 命令前检查 `$NGINX_CONTAINER` 是否运行

- upstream 文件写入使用原子操作（先写 .tmp 再 mv），防止 nginx 读到半写文件
- `/opt/noda/` 目录在 init 和 start 中用 `sudo mkdir -p` 创建，状态文件用 `sudo tee` 写入
- 脚本可被 source（函数定义不触发执行），支持 Phase 22 通过 source 复用函数（D-03）

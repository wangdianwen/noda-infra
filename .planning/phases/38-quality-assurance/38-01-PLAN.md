按照以下步骤依次执行：

**步骤 1 — 安装 shfmt**
运行 `brew install shfmt`，然后用 `shfmt --version` 确认安装成功。

**步骤 2 — 创建 .editorconfig（per D-03, D-04）**
在项目根目录创建 `.editorconfig`，内容如下：

```ini
# .editorconfig
# https://editorconfig.org
root = true

[*]
end_of_line = lf
charset = utf-8
insert_final_newline = true
trim_trailing_whitespace = true

[*.sh]
indent_style = space
indent_size = 4
shell_variant = bash
switch_case_indent = true
function_next_line = true

# 排除 DEPRECATED 脚本（per D-05）
[scripts/deploy/deploy-findclass-zero-deps.sh]
ignore = true
```

关键配置映射（per D-03）：
- `indent_style = space` + `indent_size = 4` 对应 `-i 4`
- `switch_case_indent = true` 对应 `-ci`
- `function_next_line = true` 对应 `-fn`
- `shell_variant = bash` 确保 shfmt 使用 bash 方言（per D-03）

**步骤 3 — 创建 .shellcheckrc（per D-01, D-02）**
在项目根目录创建 `.shellcheckrc`，内容如下：

```bash
# .shellcheckrc — Noda 基础设施项目级 ShellCheck 配置
# 文档: https://github.com/koalaman/shellcheck/blob/master/shellcheck.1.md

# Shell 方言
shell=bash

# 允许跟踪 source 的外部文件
external-sources=true

# Source 搜索路径（ShellCheck 查找 source 目标文件的路径）
source-path=SCRIPTDIR
source-path=SCRIPTDIR/../lib
source-path=SCRIPTDIR/../../lib

# SC2034: 变量"未使用"
# 原因: constants.sh 和 lib/ 文件定义的常量/变量由 source 的调用方使用
# ShellCheck 无法跨文件追踪变量使用，导致 50 处误报
disable=SC2034

# SC2155: declare 和 assign 分开
# 原因: 项目中广泛使用 `local var=$(cmd)` 模式（50 处）
# 虽然可能掩盖返回值，但这是 bash 社区常见模式，逐文件修改风险大于收益（per D-02）
disable=SC2155
```

**步骤 4 — 验证配置生效**
1. `shfmt -f scripts/ | wc -l` 确认 shfmt 能识别到 60 个文件（排除 DEPRECATED 的 1 个）
2. `shellcheck scripts/**/*.sh` 确认 .shellcheckrc 生效后 warning 数量减少（SC2034 和 SC2155 应消失）
3. `shellcheck -S error scripts/**/*.sh` 确认仍然 0 error

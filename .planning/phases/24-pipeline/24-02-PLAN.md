在 `jenkins/Jenkinsfile` 中做以下修改：

1. **更新文件头部注释**：将 `8 阶段` 改为 `9 阶段`，更新阶段列表添加 CDN Purge：

```groovy
// 9 阶段: Pre-flight -> Build -> Test -> Deploy -> Health Check -> Switch -> Verify -> CDN Purge -> Cleanup
// Phase 23: Pipeline 集成与测试门禁
// Phase 24: CDN 缓存清除增强
```

2. **在 Verify stage 之后、Cleanup stage 之前插入新 stage**（per D-17, D-18）：

```groovy
        stage('CDN Purge') {
            steps {
                withCredentials([
                    string(credentialsId: 'cf-api-token', variable: 'CF_API_TOKEN'),
                    string(credentialsId: 'cf-zone-id', variable: 'CF_ZONE_ID')
                ]) {
                    sh '''
                        source scripts/lib/log.sh
                        source scripts/pipeline-stages.sh
                        pipeline_purge_cdn
                    '''
                }
            }
        }
```

3. **确保 sh 块使用单引号**（`'''`），防止 Groovy 插值泄露 `CF_API_TOKEN` 和 `CF_ZONE_ID` 环境变量值到构建日志中（per Phase 23 安全模式：单引号 sh 防止 Groovy 字符串插值）。

设计要点：
- CDN Purge 位于 Verify 后、Cleanup 前（per D-08：部署验证通过后才清 CDN）
- withCredentials 注入 `CF_API_TOKEN` 和 `CF_ZONE_ID`（per D-07/D-18）
- credentialsId 分别为 `cf-api-token` 和 `cf-zone-id`（per D-07）
- Jenkins Credentials 需要管理员在 Jenkins UI 手动创建（一次性操作，不在本 Plan 自动化范围内）
- Pre-flight 阶段不需要修改（备份检查已在 Plan 01 中集成到 pipeline_preflight 函数内部，per D-19）

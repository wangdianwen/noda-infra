创建文件 scripts/jenkins/init.groovy.d/06-matrix-auth.groovy，内容如下：

1. **插件安装检查**（复用 05-audit-trail.groovy 模式）：
   - 检查 `pm.getPlugin('matrix-auth')` 是否已安装
   - 未安装时通过 `uc.getPlugin('matrix-auth').deploy(true)` 安装，然后 `return`（需重启）
   - 已安装时继续配置权限矩阵

2. **创建 developer 用户**（per CONTEXT.md D-02）：
   - 使用 HudsonPrivateSecurityRealm 检查 developer 用户是否存在
   - 不存在则通过 `realm.createAccount('developer', 'changeme-immediately')` 创建
   - 打印提醒：管理员需通过 UI 修改初始密码

3. **配置权限矩阵**（per CONTEXT.md D-01, D-02, D-03）：
   - 使用 `new GlobalMatrixAuthorizationStrategy()` 创建策略
   - Admin 角色（per D-01）：
     - `strategy.add(Jenkins.ADMINISTER, "admin")` — Overall/Administer 全权限
   - Developer 角色（per D-02）：
     - `strategy.add(Jenkins.READ, "developer")` — Overall/Read（必须，否则其他权限无效）
     - `strategy.add(hudson.model.Item.READ, "developer")` — Job/Read 查看构建历史
     - `strategy.add(hudson.model.Item.BUILD, "developer")` — Job/Build 触发 Pipeline
     - `strategy.add(hudson.model.Item.DISCOVER, "developer")` — Job/Discover
     - `strategy.add(hudson.model.Run.READ, "developer")` — Run/Read 查看 Console Output
     - `strategy.add(hudson.model.View.READ, "developer")` — View/Read 查看视图列表
   - **不授予以下权限**（per D-03）：
     - Item/Configure（修改 Job 配置）
     - Item/Create, Item/Delete
     - Run/Delete, Run/Update
     - Credentials/*
     - Overall/Administer, Overall/Manage
     - Script Console（由 Overall/Administer 控制）
   - `instance.setAuthorizationStrategy(strategy)` + `instance.save()`

4. **幂等性**：每次执行都重新创建策略对象并设置，确保配置一致

脚本头部注释说明：执行时机（06 在字母顺序晚于 05）、幂等性、两角色权限说明。

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Groovy 脚本 → Jenkins 运行时 | init.groovy.d 脚本在 Jenkins JVM 中以最高权限执行，不受权限矩阵约束 |
| CLI → Jenkins API | jenkins-cli.jar 通过 sudo -u jenkins 执行，继承 jenkins 系统用户权限 |
| developer 用户 → Jenkins UI | 受 GlobalMatrixAuthorizationStrategy 约束，只能触发 Pipeline 和查看结果 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01 | Elevation | developer 通过 API 修改 Job 配置 | mitigate | 不授予 Item/Configure 权限（D-03），GlobalMatrixAuthorizationStrategy 强制执行 |
| T-34-02 | Tampering | Groovy 脚本被未授权修改 | mitigate | 脚本文件在 Git 版本控制中；部署后 init.groovy.d 被 setup-jenkins.sh 步骤 10 清理 |
| T-34-03 | Information Disclosure | developer 默认密码 changeme-immediately | accept | 单服务器内部环境，管理员在 Groovy 脚本执行后通过 UI 修改密码；风险低于自动化复杂度 |
| T-34-04 | Spoofing | CLI 命令未认证执行 | mitigate | 使用 sudo -u jenkins 继承系统权限，不暴露凭据文件；Jenkins 安全域已配置（01-security.groovy） |

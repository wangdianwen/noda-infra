创建 `scripts/undo-permissions.sh`，实现 D-04 最小 undo 安全网。脚本结构：

```
#!/bin/bash
set -euo pipefail
```

常量：
- `BACKUP_FILE="/opt/noda/pre-phase31-permissions-backup.txt"`
- `LOCKED_SCRIPTS` 数组包含 4 个脚本路径（与 D-03 一致）：

子命令分发（case 语句）：
1. `backup` — 调用 `backup_current_state()`
2. `undo` — 调用 `undo_permissions()`
3. `help` 或 `*` — 显示 usage

`backup_current_state()` 函数：
- `sudo mkdir -p /opt/noda`
- 写入 BACKUP_FILE，内容包含：
  - `# Phase 31 权限备份 - $(date -Iseconds)` 头部
  - `SOCKET_GROUP=$(stat -c '%G' /var/run/docker.sock 2>/dev/null || echo 'unknown')`
  - `SOCKET_MODE=$(stat -c '%a' /var/run/docker.sock 2>/dev/null || echo 'unknown')`
  - 每个锁定脚本的 `FILE_PERMS=$(stat -c '%a:%U:%G' "$script") FILE=$script`
  - systemd override 内容（`cat /etc/systemd/system/docker.service.d/*.conf 2>/dev/null || echo '无 override'`）
  - jenkins 组信息（`groups jenkins`）
- `sudo chmod 600 "$BACKUP_FILE"`

`undo_permissions()` 函数：
- 检查 BACKUP_FILE 存在，不存在则报错退出
- 恢复 socket：`sudo chown root:docker /var/run/docker.sock && sudo chmod 660 /var/run/docker.sock`
- 移除 systemd override：`sudo rm -f /etc/systemd/system/docker.service.d/socket-permissions.conf`
- `sudo systemctl daemon-reload`
- `sudo systemctl restart docker`
- 恢复脚本权限：对 4 个锁定脚本执行 `sudo chmod 755 "$script" && sudo chown "$(whoami):$(whoami)" "$script"`
- 将 jenkins 重新加入 docker 组：`sudo usermod -aG docker jenkins`
- `sudo systemctl restart jenkins`
- 输出 "Phase 31 权限变更已回滚"

`usage()` 函数：
```
用法: undo-permissions.sh <命令>
命令:
  backup  备份当前权限状态（在执行权限修改前运行）
  undo    从备份恢复权限状态
  help    显示此帮助信息
```

source `scripts/lib/log.sh`（使用与 setup-jenkins.sh 相同的日志库）。

注意事项：
- 脚本需要在 macOS（开发）和 Linux（生产）都能运行 `stat` 命令。macOS `stat` 不支持 `-c` 格式，使用条件判断：`if [[ "$(uname)" == "Darwin" ]]; then stat -f '%Su:%Sg' "$f"; else stat -c '%a:%U:%G' "$f"; fi`
- 确保脚本有 `#!/bin/bash` shebang

修改 `scripts/setup-jenkins.sh` 的三个位置：

**位置 1：cmd_install 步骤 8（第 158-161 行）**

将：
```bash
# 步骤 8/10: Docker 权限
log_info "步骤 8/10: 配置 Docker 权限"
sudo usermod -aG docker jenkins
log_success "jenkins 用户已加入 docker 组"
```

替换为：
```bash
# 步骤 8/10: Docker 权限（通过 socket 属组，不加入 docker 组）
log_info "步骤 8/10: 配置 Docker 权限（socket 属组方式）"
# 确保不在 docker 组（幂等操作）
sudo gpasswd -d jenkins docker 2>/dev/null || true
# 配置 systemd override 确保 socket 属组为 jenkins
DOCKER_OVERRIDE_DIR="/etc/systemd/system/docker.service.d"
sudo mkdir -p "$DOCKER_OVERRIDE_DIR"
sudo tee "$DOCKER_OVERRIDE_DIR/socket-permissions.conf" > /dev/null <<'OVERRIDE'
[Service]
ExecStartPost=/bin/sh -c 'chown root:jenkins /var/run/docker.sock && chmod 660 /var/run/docker.sock'
OVERRIDE
sudo systemctl daemon-reload
# 立即应用（如果 Docker 正在运行）
if systemctl is-active --quiet docker 2>/dev/null; then
    sudo systemctl restart docker
fi
log_success "Docker socket 权限配置完成（jenkins 用户通过 socket 属组访问）"
```

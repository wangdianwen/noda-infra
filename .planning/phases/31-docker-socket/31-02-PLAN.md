子命令分发（case 语句）：`apply`, `verify`, `hook`, `help`/`*`

**`cmd_apply()` 函数：** 执行所有权限配置（幂等）
1. 输出 "Phase 31: 应用权限配置..."
2. 创建 /opt/noda/ 目录：`sudo mkdir -p /opt/noda && sudo chown root:jenkins /opt/noda && sudo chmod 770 /opt/noda`（确保 manage-containers.sh 可写 active-env）
3. **文件权限锁定**（D-03）：
   - 遍历 LOCKED_SCRIPTS 数组
   - 对每个文件：`sudo chown root:jenkins "$script" && sudo chmod 750 "$script"`
   - 输出每个文件的权限确认
4. **systemd override**（PERM-02）：
   - `sudo mkdir -p "$DOCKER_OVERRIDE_DIR"`
   - 写入 `$DOCKER_OVERRIDE_CONF`：
     ```ini
     [Service]
     ExecStartPost=/bin/sh -c 'chown root:jenkins /var/run/docker.sock && chmod 660 /var/run/docker.sock'
     ```
   - `sudo systemctl daemon-reload`
5. **立即应用 socket 权限**：
   - `if systemctl is-active --quiet docker 2>/dev/null; then sudo systemctl restart docker; fi`
6. **从 docker 组移除 jenkins**（如果存在）：
   - `sudo gpasswd -d jenkins docker 2>/dev/null || true`
7. **重启 Jenkins**（组变更需要进程重启）：
   - `if systemctl is-active --quiet jenkins 2>/dev/null; then sudo systemctl restart jenkins; fi`
8. **调用 `cmd_hook`** 创建 post-merge hook
9. 输出 "Phase 31: 权限配置完成"

**`cmd_verify()` 函数：** 验证所有权限配置
1. `local all_ok=true`
2. 验证 socket 属组：`stat -c '%G' /var/run/docker.sock` 等于 `jenkins`（macOS 上用 `stat -f '%Sg'`）
3. 验证 socket 权限：`stat -c '%a'` 为 `660`
4. 验证 systemd override 文件存在
5. 遍历 LOCKED_SCRIPTS 验证权限：
   - `stat -c '%a:%U:%G'` 应为 `750:root:jenkins`（macOS 上用 `stat -f '%Lp:%Su:%Sg'`）
6. 验证 post-merge hook 存在且可执行
7. 验证 jenkins 用户可执行 docker：`sudo -u jenkins docker info >/dev/null 2>&1`
8. 验证 /opt/noda/ 目录权限：`stat -c '%a:%U:%G'` 应为 `770:root:jenkins`
9. 输出 PASS/FAIL 汇总

**`cmd_hook()` 函数：** 创建 Git post-merge hook（D-02）
1. `HOOK_FILE="$PROJECT_ROOT/.git/hooks/post-merge"`
2. 写入 hook 内容：
   ```bash
   #!/bin/bash
   # .git/hooks/post-merge — git pull 后恢复部署脚本锁定权限
   # 由 apply-file-permissions.sh 创建，不在版本控制中

   PROJECT_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"

   LOCKED_SCRIPTS=(
       "$PROJECT_ROOT/scripts/deploy/deploy-apps-prod.sh"
       "$PROJECT_ROOT/scripts/deploy/deploy-infrastructure-prod.sh"
       "$PROJECT_ROOT/scripts/pipeline-stages.sh"
       "$PROJECT_ROOT/scripts/manage-containers.sh"
   )

   for script in "${LOCKED_SCRIPTS[@]}"; do
       if [ -f "$script" ]; then
           chown root:jenkins "$script" 2>/dev/null || \
               sudo chown root:jenkins "$script" 2>/dev/null || true
           chmod 750 "$script" 2>/dev/null || \
               sudo chmod 750 "$script" 2>/dev/null || true
       fi
   done
   ```
3. `chmod +x "$HOOK_FILE"`
4. 输出 "post-merge hook 已创建: $HOOK_FILE"

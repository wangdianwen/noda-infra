**1. 修复 theme.properties（per 13-RESEARCH.md Pitfall 1 + 13-UI-SPEC.md Prerequisite Fix）**

将 theme.properties 内容替换为：
```
parent=keycloak
import=common/keycloak
styles=css/styles.css
```

仅添加 `styles=css/styles.css` 这一行。这告诉 Keycloak 加载自定义 CSS 文件。缺少此行会导致 CSS 文件存在但不被加载（Pitfall 1）。

**2. 填充 styles.css（per 13-UI-SPEC.md CSS Override Contract + 13-RESEARCH.md Code Examples）**

```css
/* ============================================
   Noda Login Theme - Brand Color Overrides
   ============================================
   品牌色：Pounamu Green #0D9B6A（主色）
   设计系统：noda-apps/packages/design-tokens
   策略：最小覆盖，仅修改颜色变量和关键选择器（D-05, D-06）

/* --- PatternFly 全局变量覆盖（:root 级别）--- */
:root {
    /* 主色：Pounamu Green #0D9B6A（D-01） */
    --pf-global--primary-color--100: #0D9B6A;
    --pf-global--primary-color--200: #0a7d55;
    --pf-global--primary-color--light-100: #0D9B6A;

/* --- 直接选择器覆盖（login.css 硬编码值）--- */

/* 登录卡片顶部边框条 */

#kc-header-wrapper {
    color: #0f172a;
}

/* 链接 hover 色（login.css 硬编码 #0099d3） */
.login-pf a:hover {
    color: #0D9B6A;
}

/* 认证方式选择标题 */
.select-auth-box-headline {
    color: #0D9B6A;
}

/* 输入框焦点环（D-01: Pounamu Green） */
input:focus, select:focus {
    border-color: #0D9B6A !important;
    box-shadow: 0 0 0 2px rgba(13, 155, 106, 0.25) !important;
}

/* 按钮圆角（D-03: 0.5rem） */
.pf-c-button {
    border-radius: 0.5rem;
}

/* 输入框边框色和圆角（D-02: Slate 200, D-03: 0.5rem） */
#kc-form input[type="text"],
#kc-form input[type="password"] {
    border-color: #e2e8f0;
    border-radius: 0.5rem;
}
```

5. 确保 Keycloak Admin Console 中 noda realm 的 Login Theme 已设置为 `noda`。如果未设置，通过 curl 调用 Keycloak Admin REST API 设置：
```bash
# 获取 admin token
TOKEN=$(curl -s http://localhost:18080/realms/master/protocol/openid-connect/token \
  -d client_id=admin-cli \
  -d username=admin \
  -d password=admin \
  | jq -r .access_token)

# 获取当前 realm 配置
CURRENT_THEME=$(curl -s http://localhost:18080/admin/realms/noda \
  -H "Authorization: Bearer $TOKEN" | jq -r '.loginTheme // "null"')

# 如果不是 noda，则更新
if [ "$CURRENT_THEME" != "noda" ]; then
  curl -s -X PUT http://localhost:18080/admin/realms/noda \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{"loginTheme": "noda"}'
  echo "Login theme updated to noda"
else
  echo "Login theme already set to noda"
fi
```

注意：keycloak-dev 默认 admin 密码可能不同，检查 docker-compose.dev.yml 中的 KEYCLOAK_ADMIN_PASSWORD 环境变量。如果 admin/admin 不行，则需要用户手动在 Admin Console 中设置主题。

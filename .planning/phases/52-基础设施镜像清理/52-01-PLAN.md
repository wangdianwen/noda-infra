**Stage 1: 构建阶段（per D-01, D-02）：**
```dockerfile
# --- Stage 1: 构建阶段 - 下载外部二进制 ---
FROM alpine:3.21 AS builder

# 安装构建时下载工具
RUN apk add --no-cache wget gnupg

# 下载 cloudflared 二进制
RUN ARCH=$(uname -m) && \
    if [ "$ARCH" = "x86_64" ]; then CLOUDFLARED_ARCH="amd64"; \
    elif [ "$ARCH" = "aarch64" ]; then CLOUDFLARED_ARCH="arm64"; \
    else echo "Unsupported architecture: $ARCH" && exit 1; fi && \
    wget -q "https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-${CLOUDFLARED_ARCH}" \
         -O /usr/local/bin/cloudflared && \
    chmod +x /usr/local/bin/cloudflared && \
    cloudflared --version

# 下载 Doppler CLI（通过官方 apk 仓库 + GPG key 验证）
RUN wget -q -t3 'https://packages.doppler.com/public/cli/rsa.8004D9FF50437357.key' \
       -O /etc/apk/keys/cli@doppler-8004D9FF50437357.rsa.pub && \
    echo 'https://packages.doppler.com/public/cli/alpine/any-version/main' | \
      tee -a /etc/apk/repositories && \
    apk add doppler

# 验证二进制路径（确保 COPY 目标正确）
RUN ls -la /usr/local/bin/cloudflared /usr/bin/doppler
```

注意：Doppler 使用官方 apk 仓库 + GPG 验证方式安装（与现有 Dockerfile 一致）。apk 安装的 doppler 二进制位于 `/usr/bin/doppler`。builder 阶段末尾的 `ls -la` 验证命令确保路径正确，如果路径不同，构建会失败（快速发现问题）。

**Stage 2: 运行时阶段（per D-03, D-04, D-05, D-06, D-07）：**
```dockerfile
# --- Stage 2: 运行时阶段 - 最小依赖 ---
FROM alpine:3.21

# 仅安装运行时必需的包（无 wget/gnupg/curl，per D-03, D-04）
RUN apk add --no-cache \
    bash \
    jq \
    coreutils \
    rclone \
    dcron \
    supervisor \
    ca-certificates \
    postgresql17-client \
    age

# 从构建阶段传递二进制文件（per D-02）
COPY --from=builder /usr/local/bin/cloudflared /usr/local/bin/cloudflared
COPY --from=builder /usr/bin/doppler /usr/bin/doppler
```

注意：
- 运行时包列表包含 D-05 (jq)、D-06 (coreutils)、D-07 (bash, rclone, dcron, supervisor, ca-certificates, postgresql17-client, age) 中指定的所有包
- 不含 curl (per D-04)、wget、gnupg
- 使用 `--no-cache` 而非 `rm -rf /var/cache/apk/*`（`--no-cache` 已阻止缓存写入，per RESEARCH A4）

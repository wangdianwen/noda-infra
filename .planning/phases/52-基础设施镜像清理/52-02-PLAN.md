```dockerfile
# ============================================
# Noda 数据库备份系统 - Docker 服务容器
# ============================================
# 功能：独立的备份服务容器，内部运行 cron 定时任务
# 镜像名: noda-backup:latest
# 容器名: opdev
# ============================================

FROM postgres:17-alpine

# 合并为单个 RUN：安装工具 + 创建目录 + 设置权限
# 注意：移除 curl（脚本中未使用），per D-08
RUN apk add --no-cache \
    bash \
    jq \
    coreutils \
    rclone \
    dcron \
    ca-certificates \
    && mkdir -p /var/log/noda-backup /app/history \
    && touch /var/log/noda-backup/backup.log \
             /var/log/noda-backup/test.log \
    && chmod 666 /var/log/noda-backup/*.log

# 设置工作目录
WORKDIR /app

# 复制备份脚本
COPY scripts/backup/ /app/

# 复制 crontab 配置
COPY deploy/crontab /etc/crontabs/root

# 复制启动脚本
COPY deploy/entrypoint.sh /app/entrypoint.sh
RUN chmod +x /app/entrypoint.sh

# 设置环境变量
ENV BACKUP_DIR=/tmp/postgres_backups \
    HISTORY_DIR=/app/history \
    SCRIPT_DIR=/app

# 暴露端口（如果需要健康检查）
EXPOSE 8080

# 启动 cron 服务
CMD ["/app/entrypoint.sh"]
```

关键变更说明：
1. **4 RUN → 2 RUN**：RUN 1 (apk add) + RUN 3 (mkdir+touch+chmod) + RUN 4 (mkdir /app/history) 合并为单个 RUN。RUN 2 (chmod +x entrypoint.sh) 必须保留在 COPY 之后，不能合并在 apk add 同一个 RUN 中。
2. **移除 curl**：backup 脚本中 grep 确认无 curl 调用。健康检查使用 pg_isready（由 postgres:17-alpine 基础镜像提供）。
3. **mkdir 合并**：`mkdir -p /var/log/noda-backup` 和 `mkdir -p /app/history` 合并为 `mkdir -p /var/log/noda-backup /app/history`。
4. **COPY 和 ENTRYPOINT 保持不变**（per D-09）。

  <what-built>
Plan 43-01 创建了 scripts/lib/cleanup.sh 综合清理库（9 个清理函数 + 2 个 wrapper + 磁盘快照）。
Plan 43-02 修改了 scripts/pipeline-stages.sh，在 pipeline_cleanup/pipeline_infra_cleanup 末尾调用 cleanup wrapper，在 pipeline_deploy/pipeline_infra_deploy 开头调用 disk_snapshot("部署前")。
  </what-built>
  <how-to-verify>
**验证步骤（按优先级排列）：**

1. **触发 findclass-ssr Pipeline（最全面验证）**
   - 访问 Jenkins UI: http://\<server-ip\>:8888
   - 点击 `findclass-ssr-deploy` 任务 -> "Build Now"
   - 等待构建完成
   - 检查构建日志中 Cleanup 阶段的输出

2. **检查 findclass-ssr 构建日志中应包含：**
   - Deploy 阶段开头: `=== 磁盘快照: 部署前 ===`
   - Cleanup 阶段末尾: 清理输出（build cache、containers、networks、volumes、node_modules、temp files）
   - Cleanup 阶段最后: `=== 磁盘快照: 清理后 ===`
   - 对比两个快照中 "宿主机: XX% 已用"，清理后应略低或持平

3. **验证安全红线：**
   ```bash
   docker volume ls | grep postgres_data
   ```
   postgres_data 卷必须仍然存在

4. **（可选）触发其他 Pipeline 验证：**
   - keycloak-deploy Pipeline: 验证无 build cache 清理（keycloak 用官方镜像）
   - infra Pipeline (noda-ops): 验证 build cache 清理 + 备份文件清理
   - infra Pipeline (postgres): 验证备份文件清理（无 build cache）

5. **检查要点：**
   - 每个 Pipeline 构建日志中有且仅有 2 次 disk_snapshot（部署前 + 清理后）
   - 清理函数输出无错误（docker prune 命令应正常执行）
   - 清理后磁盘用量应持平或降低（不会变高）
   - 正在运行的服务容器未被误删
  </how-to-verify>

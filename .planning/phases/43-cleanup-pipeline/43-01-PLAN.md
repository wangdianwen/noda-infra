
**4. 清理函数（9 个，按 CLEANUP-RESEARCH.md 第十一章模板实现）：**

(a) `cleanup_docker_build_cache()` -- DOCK-01
- 参数: `$1` = 保留小时数（默认 `$BUILD_CACHE_RETENTION_HOURS`，即 24）
- 命令: `docker buildx prune -f --filter "until=${retention_hours}h" 2>/dev/null || true`
- 日志: 清理前用 `docker buildx du 2>/dev/null | tail -1` 获取缓存大小，清理后记录
- 幂等: `|| true` 确保失败不传播

(b) `cleanup_dangling_images()` -- 补充清理
- 无位置参数
- 命令: 先 `docker images -f "dangling=true" -q 2>/dev/null | grep -c .` 计数，大于 0 才执行 `docker image prune -f 2>/dev/null || true`
- 日志: 记录清理的 dangling images 数量

(c) `cleanup_stopped_containers()` -- DOCK-02
- 参数: `$1` = 保留小时数（默认 `$CONTAINER_RETENTION_HOURS`，即 24）
- 命令: `docker container prune -f --filter "until=${retention_hours}h" 2>/dev/null || true`
- 安全: 只清理已停止容器，`--filter until` 保护最近停止的容器

(d) `cleanup_unused_networks()` -- 补充清理
- 无位置参数
- 命令: `docker network prune -f 2>/dev/null || true`
- 安全: `docker network prune` 只删除未使用的匿名/自定义网络，不删除 external 网络（noda-network 安全）

(e) `cleanup_anonymous_volumes()` -- DOCK-03（核心安全红线）
- 无位置参数
- 命令: `docker volume prune -f 2>/dev/null || true`
- **绝对不加 `--all` 标志** -- 只清理匿名卷，保护 postgres_data 瑞名卷
- 日志中明确标注 "安全：不删除命名卷（如 postgres_data）"

(f) `cleanup_node_modules()` -- CACHE-01
- 参数: `$1` = workspace 路径（必须提供，为空则 return 0）
- 目标: `$workspace/noda-apps/node_modules`
- 使用 `du -sh "$target" 2>/dev/null | awk '{print $1}'` 获取大小
- 使用 `rm -rf "$target"` 删除
- 目录不存在时 log_info "无 node_modules 需要清理" 并 return

(g) `cleanup_jenkins_temp_files()` -- FILE-02
- 参数: `$1` = workspace 路径（默认 `$WORKSPACE`）
- 删除文件列表:
  - `$workspace/deploy-failure-container.log`
  - `$workspace/deploy-failure-nginx.log`
  - `$workspace/deploy-failure-infra.log`
- 额外: `find "$workspace" -name "*.tmp" -type f -delete 2>/dev/null || true`
- 所有删除操作加 `2>/dev/null || true`

(h) `cleanup_old_infra_backups()` -- FILE-01
- 参数: `$1` = 服务名（必须提供），`$2` = 保留天数（默认 `$BACKUP_RETENTION_DAYS`，即 30）
- 备份目录: `${BACKUP_HOST_DIR:-$PROJECT_ROOT/docker/volumes/backup}/infra-pipeline/${service}`
- 目录不存在时 return 0
- 命令: `find "$backup_dir" -name "*.sql.gz" -type f -mtime +"${retention_days}" -print -delete 2>/dev/null || true`
- 日志: 记录删除的文件数量（通过管道计数 find 输出行数）

(i) `disk_snapshot()` -- DOCK-04（per D-05，纯文本输出到日志）
- 参数: `$1` = 标签（如 "部署前" / "清理后"）
- 输出格式:
  ```
  === 磁盘快照: {label} ===
  宿主机: {used_pct} 已用 ({used}/{total})
  Docker: {docker system df 的前 5 行}
  ```
- 宿主机: `df -h / | awk 'NR==2{print $5 " 已用 (" $3 "/" $2 ")"}'`
- Docker: `docker system df 2>/dev/null | head -5 || echo "Docker: 不可用"`
- 末尾空行分隔

**5. 高层 Wrapper 函数（per D-03/D-04）：**

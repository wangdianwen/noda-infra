Output: 修改后的 Dockerfile.noda-ops（Alpine 3.21 + postgresql17-client）和 docker-compose.yml（PGSSLMODE=disable 环境变量）。
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| noda-ops 容器 → postgres 容器 | PG 客户端连接跨 Docker 内部网络，sslmode=disable 意味着明文传输 |
| Docker 内部网络 → 外部 | noda-network 不暴露到宿主机端口，外部无法嗅探 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Information Disclosure | noda-ops → postgres (sslmode=disable) | accept | Docker 内部网络（noda-network）仅容器可达，宿主机外部无法嗅探。PGSSLMODE=disable 仅影响 Docker 内部网络通信，不涉及公网传输。网络层由 Docker bridge 隔离保护。 |
| T-15-02 | Tampering | pg_dump 版本不匹配导致备份损坏 | mitigate | PG-01：升级到 postgresql17-client 17.9，确保客户端与服务端主版本一致（17.x），消除版本不匹配导致的数据损坏风险。Alpine 3.21 提供 postgresql17-client 17.9-r0。 |
| T-15-03 | Information Disclosure | 备份数据传输明文 | accept | 备份数据通过 Docker 内部网络传输，不在公网暴露。备份文件存储在 B2 云存储时由 rclone 使用 TLS 加密上传。 |
| T-15-04 | Spoofing | PGSSLMODE 环境变量被恶意修改 | accept | docker-compose.yml 纳入版本控制，变更需要 git commit。无运行时动态修改 PGSSLMODE 的路径。 |
</threat_model>

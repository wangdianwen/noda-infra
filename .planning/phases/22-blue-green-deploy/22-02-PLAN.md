- 回滚场景下健康检查参数可以更激进（10 次 x 3 秒 = 30 秒，而非 120 秒）

**函数 1: http_health_check(container_name, [max_retries], [interval])**
独立定义（不复用 blue-green-deploy.sh），逻辑相同但回滚场景参数不同：
- 默认 10 次重试，间隔 3 秒（30 秒超时，回滚需要快速）
- 使用 BusyBox wget：`docker exec "$container" wget --quiet --tries=1 --spider "http://localhost:3001/api/health"`
- 成功 return 0，失败 log_error + 打印容器日志 return 1

**函数 2: e2e_verify(target_env, [max_retries], [interval])**
独立定义，逻辑与 blue-green-deploy.sh 相同：
- 默认 5 次重试，间隔 2 秒
- 通过 nginx 容器 curl 目标容器名
- 优先 curl，不可用时回退 wget（与 Plan 01 相同的检测逻辑）
- HTTP 200 为成功

**主函数 main()：**

per D-07/D-08：回滚流程是验证旧容器运行中 -> 更新 upstream -> reload nginx -> 更新 active-env。

```bash
main() {
  local active_env
  active_env=$(get_active_env)

  # 回滚目标 = 非活跃环境
  local rollback_env
  if [ "$active_env" = "blue" ]; then
    rollback_env="green"
  else
    rollback_env="blue"
  fi

  local rollback_container
  rollback_container=$(get_container_name "$rollback_env")

  log_info "=========================================="
  log_info "紧急回滚"
  log_info "=========================================="
  log_info "当前活跃: $active_env ($(get_container_name "$active_env"))"
  log_info "回滚目标: $rollback_env ($rollback_container)"
  log_info ""
```

```bash
    exit 1
  fi
```

**步骤 3：验证回滚目标健康**
```bash
  log_info "步骤 1/4: 验证回滚目标容器健康"
  if ! http_health_check "$rollback_container" 10 3; then
    log_error "回滚目标容器 $rollback_container 不健康，拒绝回滚"
    log_error "请手动检查容器状态: docker logs $rollback_container --tail 50"
    exit 1
  fi
```

**步骤 4：切换流量**（per D-08）
```bash
  log_info "步骤 2/4: 切换流量 $active_env -> $rollback_env"
  update_upstream "$rollback_env"

  if ! docker exec "$NGINX_CONTAINER" nginx -t; then
    log_error "nginx 配置验证失败，恢复 upstream"
    update_upstream "$active_env"
    exit 1
  fi

  reload_nginx
  set_active_env "$rollback_env"
```

**步骤 5：E2E 验证**
```bash
  log_info "步骤 3/4: E2E 验证"
  if ! e2e_verify "$rollback_env"; then
    log_error "回滚后 E2E 验证失败"
    log_error "流量已切换到 $rollback_env，请手动检查服务状态"
    exit 1
  fi
```

**步骤 6：完成**
```bash
  log_info "步骤 4/4: 完成"
  log_success "=========================================="
  log_success "回滚完成"
  log_success "=========================================="
  log_success "$active_env -> $rollback_env"
  log_info ""
  log_info "新版本容器 $(get_container_name "$active_env") 仍在运行"
  log_info "确认无误后可手动停止: docker stop -t 30 $(get_container_name "$active_env")"
}

main "$@"
```

- 回滚脚本是独立的，不 source blue-green-deploy.sh（紧急工具必须独立可运行）
- 回滚时健康检查更激进（30 秒 vs 120 秒），因为旧容器通常已经运行过，不需要冷启动

#!/bin/bash
set -euo pipefail

**常量定义：**

- `AUDITD_CONF="/etc/audit/auditd.conf"`
- `BACKUP_DIR="/opt/noda/pre-phase33-backup"`

**平台检测（复用 detect_platform 函数）：**
```bash
detect_platform() {
    local os
    os="$(uname)"
    if [[ "$os" == "Darwin" ]]; then echo "macos"
    else echo "linux"
    fi
}
PLATFORM="$(detect_platform)"
```

**cmd_install() 实现步骤（per AUDIT-01, AUDIT-02, D-01）：**

1. **平台检查：** macOS 跳过（`log_warn` + `return 0`）

2. **安装 auditd 包：** 如果 `dpkg -l auditd` 失败，执行 `apt install -y auditd audispd-plugins`

3. **检查并移除 Debian no-audit 默认规则（RESEARCH.md Pitfall 1）：**
   - 如果 `/etc/audit/rules.d/10-no-audit.rules` 存在，执行 `rm -f` 删除
   - 输出 `log_info` 说明已移除

4. **备份当前规则文件：** 如果 `$RULES_FILE` 存在，复制到 `$BACKUP_DIR/noda-docker.rules.bak`

5. **写入 auditd 规则文件（heredoc 方式）：**
```bash
tee "$RULES_FILE" > /dev/null <<'EOF'
## Noda Docker Command Audit Rules (Phase 33, AUDIT-01)
## 监控所有 docker 命令执行，记录 auid/时间/命令参数

# 删除已有 docker-cmd 规则（幂等）
-D -k docker-cmd

# 监控 docker 命令执行（普通用户 auid >= 1000）
-a always,exit -F arch=b64 -S execve -F exe=/usr/bin/docker -F auid>=1000 -F auid!=-1 -k docker-cmd

# 监控 jenkins 系统用户的 docker 命令（auid 可能为 unset，per RESEARCH.md Pitfall 6）
-a always,exit -F arch=b64 -S execve -F exe=/usr/bin/docker -F uid=jenkins -k docker-cmd
EOF
```

6. **设置规则文件权限：** `chmod 0640 "$RULES_FILE"` + `chown root:root "$RULES_FILE"`

7. **修改 auditd.conf（per AUDIT-02, D-01）：** 使用 sed 修改以下参数：
   - `log_group = root`（确保日志 root 只读）
   - `max_log_file = 50`（单文件最大 50MB）
   - `max_log_file_action = ROTATE`（达到上限轮转）
   - `num_logs = 30`（保留 30 个文件，约 30 天）
   - `log_format = ENRICHED`（包含用户名解析）
   
   每个参数用 `sed -i` 替换：`sed -i "s/^${key}*=.*/${key} = ${value}/" "$AUDITD_CONF"`，如果行不存在则追加。

8. **加载规则：** `augenrules --load`

9. **启动/重启 auditd：** `systemctl enable auditd` + `systemctl restart auditd`

10. **输出安装成功信息**（规则文件路径、备份路径、查询命令提示）

**cmd_verify() 实现步骤：**

1. **平台检查：** macOS 跳过

2. **检查规则文件存在：** `[ -f "$RULES_FILE" ]`

3. **检查规则文件权限：** `stat -c '%a:%U:%G' "$RULES_FILE"` 期望 `640:root:root`

4. **检查 auditd 服务运行：** `systemctl is-active --quiet auditd`

5. **检查规则已加载：** `auditctl -l | grep -q "docker-cmd"`

6. **检查 auditd.conf 参数：**
   - grep `log_group = root` 
   - grep `max_log_file = 50`
   - grep `max_log_file_action = ROTATE`
   - grep `num_logs = 30`
   - grep `log_format = ENRICHED`

7. **检查日志目录权限（AUDIT-02）：** `stat -c '%a:%U:%G' /var/log/audit/` 期望目录 root:root，且 audit.log 文件权限 0600 或 0640、属主 root:root

8. **汇总结果：** all_ok=true/false + PASS/FAIL 输出

**cmd_uninstall() 实现步骤：**

1. **平台检查：** macOS 跳过

2. **删除规则文件：** `rm -f "$RULES_FILE"`

3. **重新加载规则：** `augenrules --load`

4. **输出卸载成功信息**

**子命令分发（与 install-sudoers-whitelist.sh 完全一致的模式）：**
```bash
case "${1:-}" in
    install)   cmd_install ;;
    verify)    cmd_verify ;;
    uninstall) cmd_uninstall ;;
    help|--help|-h) usage ;;
    *)         usage && exit 1 ;;
esac
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 用户空间 -> 内核 | auditd 规则在内核层执行，用户态无法绕过 |
| auditd 日志文件 | /var/log/audit/ 权限 root:root，普通用户无写权限 |
| auditd 规则文件 | /etc/audit/rules.d/ 权限 root:root 0640 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Tampering | /var/log/audit/ 日志文件 | mitigate | auditd.conf log_group=root + 文件权限 0600 root:root |
| T-33-02 | Denial of Service | auditd 日志磁盘占用 | mitigate | max_log_file=50 + max_log_file_action=ROTATE + num_logs=30 (per D-01) |
| T-33-03 | Elevation of Privilege | 用户绕过审计 | mitigate | 内核级 syscall 规则，无法通过 shell alias/PATH 绕过 |
| T-33-04 | Tampering | auditd 规则文件删除 | accept | 规则文件 root:root 0640 保护；-e 2 锁定留待 Future requirement |
</threat_model>

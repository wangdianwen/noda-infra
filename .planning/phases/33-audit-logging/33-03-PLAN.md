**1. 创建 `scripts/install-sudo-log.sh`**

严格遵循 `scripts/install-sudoers-whitelist.sh` 的三件套模式。

**脚本头部（与 install-sudoers-whitelist.sh 完全一致的模式）：**
```bash
#!/bin/bash
set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"

SUDOERS_FILE="/etc/sudoers.d/noda-audit"
SUDO_LOG_DIR="/var/log/sudo-logs"
SUDO_LOG_FILE="$SUDO_LOG_DIR/sudo.log"
LOGROTATE_SRC="$PROJECT_ROOT/config/logrotate/sudo-logs"
LOGROTATE_DST="/etc/logrotate.d/sudo-logs"
BACKUP_DIR="/opt/noda/pre-phase33-backup"

source "$PROJECT_ROOT/scripts/lib/log.sh"
```

**平台检测（复用 detect_platform 函数）：**
```bash
detect_platform() {
    local os
    os="$(uname)"
    if [[ "$os" == "Darwin" ]]; then echo "macos"
    else echo "linux"
    fi
}
PLATFORM="$(detect_platform)"
```

**cmd_install() 实现步骤（per AUDIT-04, D-01）：**

1. **平台检查：** macOS 跳过（`log_warn` + `return 0`）

2. **创建 sudo 日志目录（per RESEARCH.md Pitfall 5）：**
   - `mkdir -p "$SUDO_LOG_DIR"`
   - `chmod 700 "$SUDO_LOG_DIR"`
   - `chown root:root "$SUDO_LOG_DIR"`

3. **创建初始日志文件（确保 sudo 有文件可写）：**
   - `touch "$SUDO_LOG_FILE"`
   - `chmod 0600 "$SUDO_LOG_FILE"`
   - `chown root:root "$SUDO_LOG_FILE"`

4. **备份现有 sudoers 文件（如果存在）：** 复制到 `$BACKUP_DIR/noda-audit.bak`

5. **写入 sudoers 配置文件（heredoc 方式）：**
```bash
tee "$SUDOERS_FILE" > /dev/null <<'EOF'
# Noda Sudo Audit Log Configuration (Phase 33, AUDIT-04)
# 将所有 sudo 操作记录到独立日志文件

Defaults logfile=/var/log/sudo-logs/sudo.log
EOF
```

6. **验证 sudoers 语法（关键步骤，per RESEARCH.md Pitfall 4）：**
   - `visudo -cf "$SUDOERS_FILE"`
   - 失败则删除文件并退出

7. **设置文件权限：** `chmod 0440 "$SUDOERS_FILE"` + `chown root:root "$SUDOERS_FILE"`

8. **复制 logrotate 配置（如果源文件存在）：**
   - `if [ -f "$LOGROTATE_SRC" ]; then cp "$LOGROTATE_SRC" "$LOGROTATE_DST"; fi`
   - `chmod 0644 "$LOGROTATE_DST"`

9. **输出安装成功信息**（sudoers 文件路径、日志路径、logrotate 路径、验证命令提示）

**cmd_verify() 实现步骤：**

1. **平台检查：** macOS 跳过

2. **检查 sudoers 文件存在：** `[ -f "$SUDOERS_FILE" ]`

3. **检查 sudoers 文件权限：** `stat -c '%a:%U:%G' "$SUDOERS_FILE"` 期望 `440:root:root`

4. **语法验证：** `visudo -cf "$SUDOERS_FILE" >/dev/null 2>&1`

5. **检查内容包含 logfile 配置：** `grep -q "logfile=/var/log/sudo-logs/sudo.log" "$SUDOERS_FILE"`

6. **检查日志目录存在：** `[ -d "$SUDO_LOG_DIR" ]`

7. **检查日志目录权限：** `stat -c '%a:%U:%G' "$SUDO_LOG_DIR"` 期望 `700:root:root`

8. **检查 logrotate 配置存在：** `[ -f "$LOGROTATE_DST" ]`

9. **检查 logrotate 内容：** `grep -q "rotate 14" "$LOGROTATE_DST"`

10. **汇总结果：** all_ok=true/false + PASS/FAIL 输出

**cmd_uninstall() 实现步骤：**

1. **平台检查：** macOS 跳过

2. **删除 sudoers 文件：** `rm -f "$SUDOERS_FILE"`

3. **删除 logrotate 配置：** `rm -f "$LOGROTATE_DST"`

4. **输出卸载成功信息**（日志目录和日志文件保留不删除，包含历史审计数据）

**2. 创建 `config/logrotate/sudo-logs`**

```bash
# /etc/logrotate.d/sudo-logs
# sudo 操作日志轮转配置（per AUDIT-04, D-01）
# 保留 14 天，单文件最大 50MB
/var/log/sudo-logs/sudo.log {
    daily
    rotate 14
    maxsize 50M
    compress
    delaycompress
    missingok
    notifempty
    create 0600 root root
}
```

参数说明（per D-01）：
- `daily` + `rotate 14` — 保留 14 天
- `maxsize 50M` — 单文件最大 50MB
- `create 0600 root root` — 新文件权限 root 只读
- `compress` + `delaycompress` — 压缩旧日志
- `missingok` + `notifempty` — 容错配置

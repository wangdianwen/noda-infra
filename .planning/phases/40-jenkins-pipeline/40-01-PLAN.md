  <action>
创建 scripts/lib/secrets.sh，实现 `load_secrets()` 函数。

函数逻辑（per D-03, D-04, D-05, D-10）：
1. 检查 `DOPPLER_TOKEN` 环境变量是否非空（使用 `${DOPPLER_TOKEN:-}` 判断）
2. 如果 DOPPLER_TOKEN 存在：
   a. 检查 doppler CLI 是否可用（`command -v doppler`）
   b. 如果不可用，用 log_error 输出错误提示并 return 1
   c. 执行 `doppler secrets download --no-file --format=env --project noda --config prd`（注意 config 名为 `prd`，per D-05）
   d. 将输出捕获到变量 _secrets 中（不在文件系统落盘）
   e. 检查命令退出码，失败则 log_error + return 1
   f. 使用 `set -a; eval "$_secrets"; set +a` 注入环境变量
   g. log_success 输出成功信息
3. 如果 DOPPLER_TOKEN 不存在（回退模式）：
   a. 遍历 `$PROJECT_ROOT/docker/.env` 和 `$HOME/Project/noda-infra/docker/.env` 两个路径
   b. 找到第一个存在的文件后 `set -a; source "$_env_path"; set +a`
   c. log_info 输出加载来源
   d. 如果都没找到，log_error + return 1

文件结构：
- 开头 shebang `#!/bin/bash`
- `set -euo pipefail` 错误处理
- 文件头注释说明功能和用途
- `load_secrets()` 函数定义
- 不包含 source guard（此文件是工具库，可被其他脚本直接 source）
- 此文件不需要 source log.sh，因为调用者（pipeline-stages.sh 等）已经 source 了 log.sh。但为了安全，在函数开头检查 log_info 是否可用，如果不可用则定义简单的 fallback 函数：
  ```bash
  if ! declare -f log_info >/dev/null 2>&1; then
      log_info() { echo "[INFO] $*"; }
      log_error() { echo "[ERROR] $*" >&2; }
      log_success() { echo "[OK] $*"; }
      log_warn() { echo "[WARN] $*"; }
  fi
  ```
  将这段放在 load_secrets() 函数体最前面。

**cmd_install() 子命令 — 安装 sudoers 白名单规则：**

1. 平台检查：macOS 输出警告 "macOS 不需要 sudoers 白名单（Docker Desktop 无 socket 权限问题）" 并 return 0
2. 检查 `/etc/sudoers.d/` 目录是否存在，不存在则 `mkdir -p`
3. 写入 sudoers 规则文件 `/etc/sudoers.d/noda-docker-readonly`，内容精确为：

```
# Noda Docker Read-Only Whitelist (Phase 32)
# 允许 admin/sudo 组用户通过 sudo 执行只读 docker 命令
# 白名单: ps, logs, inspect, stats, top (per D-05, BREAK-01)
# 黑名单: run, rm, exec, compose (per D-05, BREAK-02)

# 命令别名：只读 docker 命令
Cmnd_Alias DOCKER_READ_ONLY = \
    /usr/bin/docker ps, \
    /usr/bin/docker ps *, \
    /usr/bin/docker logs, \
    /usr/bin/docker logs *, \
    /usr/bin/docker inspect, \
    /usr/bin/docker inspect *, \
    /usr/bin/docker stats, \
    /usr/bin/docker stats *, \
    /usr/bin/docker top, \
    /usr/bin/docker top *

# 命令别名：写入 docker 命令（显式拒绝）
Cmnd_Alias DOCKER_WRITE = \
    /usr/bin/docker run, \
    /usr/bin/docker run *, \
    /usr/bin/docker rm, \
    /usr/bin/docker rm *, \
    /usr/bin/docker exec, \
    /usr/bin/docker exec *, \
    /usr/bin/docker compose *, \
    /usr/bin/docker stop, \
    /usr/bin/docker stop *, \
    /usr/bin/docker restart, \
    /usr/bin/docker restart *, \
    /usr/bin/docker kill, \
    /usr/bin/docker kill *, \
    /usr/bin/docker cp, \
    /usr/bin/docker cp *, \
    /usr/bin/docker create, \
    /usr/bin/docker create *, \
    /usr/bin/docker build, \
    /usr/bin/docker build *, \
    /usr/bin/docker push, \
    /usr/bin/docker push *, \
    /usr/bin/docker rmi, \
    /usr/bin/docker rmi *, \
    /usr/bin/docker network *, \
    /usr/bin/docker volume *, \
    /usr/bin/docker system *, \
    /usr/bin/docker container *, \
    /usr/bin/docker image *, \
    /usr/bin/docker swarm *, \
    /usr/bin/docker login, \
    /usr/bin/docker login *, \
    /usr/bin/docker pull, \
    /usr/bin/docker pull *

# 规则：写入命令全部拒绝
%sudo ALL = !DOCKER_WRITE

# 规则：只读命令允许（无需密码，因为只读无害）
%sudo ALL = NOPASSWD: DOCKER_READ_ONLY
```

4. 用 `visudo -cf "$SUDOERS_FILE"` 验证语法，验证失败则删除文件并退出
5. 设置文件权限 `0440 root:root`
6. 备份当前状态到 `$SUDOERS_BACKUP`（记录文件是否存在）
7. 输出安装成功信息

**cmd_verify() 子命令 — 验证 sudoers 规则：**

1. 检查 `$SUDOERS_FILE` 是否存在
2. 检查文件权限是否为 `0440 root:root`
3. 用 `visudo -cf` 验证语法
4. 检查白名单规则包含 5 个只读命令（ps、logs、inspect、stats、top）
5. 检查黑名单规则包含 docker exec（per D-04）
6. 所有检查通过输出 "PASS"，否则输出 "FAIL" 并 return 1

**cmd_uninstall() 子命令 — 卸载 sudoers 规则：**

1. 删除 `$SUDOERS_FILE`
2. 输出卸载成功信息

**子命令分发：**
```bash
case "${1:-}" in
  install)   cmd_install ;;
  verify)    cmd_verify ;;
  uninstall) cmd_uninstall ;;
  help|--help|-h) usage ;;
  *)         usage && exit 1 ;;
esac
```

**usage() 函数：** 显示帮助信息，列出 install/verify/uninstall 三个子命令和说明。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin -> sudoers | 管理员通过 sudo 获得受限 docker 命令执行权限 |
| sudoers.d 文件 | sudoers 规则文件必须 0440 root:root 防止篡改 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Elevation of Privilege | sudoers 白名单规则 | mitigate | Cmnd_Alias 严格限制子命令，通配符仅允许参数传递，不使用 `docker *` 等宽泛模式 |
| T-32-02 | Tampering | /etc/sudoers.d/noda-docker-readonly | mitigate | 文件权限 0440 root:root，仅 root 可写 |
| T-32-03 | Elevation of Privilege | docker compose 子命令 | mitigate | 黑名单包含 `docker compose *`，阻止通过 compose 执行写入操作 |
| T-32-04 | Information Disclosure | docker logs 输出 | accept | logs 为只读调试必需，日志内容不包含比容器内更多的敏感信息 |
</threat_model>
